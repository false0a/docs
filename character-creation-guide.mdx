## 1. 시작하기 전에

### 캐릭터란?

루벳의 캐릭터는 AI가 연기하는 대화 상대입니다. 캐릭터를 만들면 직접 채팅할 수 있고, 시나리오에 등장시킬 수도 있습니다. 다른 사용자에게 공개하면 누구나 해당 캐릭터와 대화할 수 있습니다.

### 콘텐츠 유형

루벳에서는 두 가지 유형의 콘텐츠를 제작할 수 있습니다.

| 유형     | 설명                                                    |
| ------ | ----------------------------------------------------- |
| 1:1 채팅 | 하나의 캐릭터와 1대1로 대화합니다. 기존의 캐릭터 제작 방식입니다.                |
| 멀티챗    | 여러 캐릭터가 함께 등장하는 그룹 대화입니다. 2~5명의 캐릭터가 하나의 채팅방에서 대화합니다. |

콘텐츠 유형은 제작 첫 단계(기본 정보)에서 선택하며, 선택에 따라 제작 단계가 달라집니다.

### 캐릭터를 만들 수 있는 곳

**제작 스튜디오** \> **새 캐릭터** 버튼을 클릭하면 캐릭터 생성 페이지로 이동합니다.

- 경로: 대시보드 좌측 메뉴 또는 상단 내비게이션에서 "제작 스튜디오"를 선택합니다.
- PC 환경을 권장합니다. 모바일에서도 접근 가능하지만, PC에서 더 편리하게 작업할 수 있습니다.

## 2. 캐릭터 제작 흐름

캐릭터 제작은 선택한 콘텐츠 유형에 따라 단계 구성이 달라집니다. 단계 사이를 자유롭게 이동할 수 있으며, 순서대로 완성할 필요는 없습니다.

### 1:1 채팅 제작 단계

| 단계 | 이름      | 필수 여부 | 설명                            |
| --- | ------- | ----- | ----------------------------- |
| 1  | 기본 정보   | 필수    | 이름, 성별, 이미지, 성격 키워드, 탐색 태그    |
| 2  | 상세 프로필  | 권장    | 나이, 좋아하는 것, 상세 소개글            |
| 3  | 인트로씬    | 필수    | 대화 시작 시 표시되는 고정 텍스트           |
| 4  | AI 설정   | 권장    | AI가 캐릭터를 연기할 때 참고하는 페르소나 프롬프트 |
| 5  | 스프라이트   | 선택    | 대화 중 표시되는 캐릭터 일러스트            |
| 6  | 호감도 설정  | 선택    | 호감도에 따른 호칭/말투 변화              |
| 7  | 검토 및 생성 | 필수    | 최종 확인, 공개 설정, 저장              |

### 멀티챗 제작 단계

| 단계 | 이름      | 필수 여부 | 설명                          |
| --- | ------- | ----- | --------------------------- |
| 1  | 기본 정보   | 필수    | 제목, 이미지, 분위기 키워드, 탐색 태그     |
| 2  | 상세 프로필  | 권장    | 상세 소개 페이지                   |
| 3  | 인트로씬    | 필수    | 대화 시작 시 표시되는 고정 텍스트         |
| 4  | 콘텐츠 설정  | 권장    | 세계관, 상황 설정, 유저 역할 등 공통 프롬프트 |
| 5  | 참여 캐릭터  | 필수    | 그룹 대화에 참여하는 캐릭터 2~5명 등록     |
| 6  | 스프라이트   | 선택    | 대화 중 표시되는 캐릭터 일러스트          |
| 7  | 검토 및 생성 | 필수    | 최종 확인, 공개 설정, 저장            |

각 단계의 완료 여부는 상단 진행 표시줄에서 초록색 점으로 확인할 수 있습니다.

## 3. 기본 정보

캐릭터의 핵심 정보를 입력하는 단계입니다.

### 콘텐츠 유형 선택

제작 페이지에 처음 진입하면 콘텐츠 유형을 선택합니다.

| 유형     | 설명                    |
| ------ | --------------------- |
| 1:1 채팅 | 하나의 캐릭터와 1대1 대화 (기본값) |
| 멀티챗    | 여러 캐릭터가 참여하는 그룹 대화    |

> **참고:** 유형을 전환하면 해당 유형에서만 사용되는 단계가 숨겨지거나 표시됩니다. 숨겨진 단계의 데이터는 메모리에 보존되므로, 다시 전환하면 이전 입력값이 복원됩니다.

### 이름 / 제목 (필수)

| 콘텐츠 유형 | 라벨         | 예시    |
| ------ | ---------- | ----- |
| 1:1 채팅 | 캐릭터 이름     | 강하윤   |
| 멀티챗    | 멀티챗 콘텐츠 제목 | 황실의 밤 |

### 대표 이미지 (권장)

캐릭터의 대표 이미지를 업로드합니다. 이미지 영역을 클릭하면 파일을 선택할 수 있습니다.

- 지원 형식: JPG, PNG, GIF, WebP, MP4, WebM
- 이미지 최대 크기: 5MB (초과 시 자동 압축)
- 영상 최대 크기: 20MB
- 비율: 5:6 (세로형)

### 성별 (1:1 채팅 전용, 필수)

캐릭터의 성별을 선택합니다. 멀티챗에서는 이 항목이 표시되지 않습니다.

| 옵션 | 설명     |
| --- | ------ |
| 남성 | 남성 캐릭터 |
| 여성 | 여성 캐릭터 |
| 기타 | 그 외    |

세 가지 중 하나를 선택해야 합니다.

### 성격 키워드 / 분위기 키워드 (선택)

캐릭터의 성격 또는 콘텐츠의 분위기를 나타내는 태그를 입력합니다.

| 콘텐츠 유형 | 라벨      | 예시                      |
| ------ | ------- | ----------------------- |
| 1:1 채팅 | 성격 키워드  | `#츤데레`, `#냉철함`, `#다정함`  |
| 멀티챗    | 분위기 키워드 | `#궁중로맨스`, `#긴장감`, `#코믹` |

- 키워드를 입력하고 Enter 키를 누르면 추가됩니다.
- 중복 키워드는 추가되지 않습니다.
- 추가된 키워드는 `#키워드` 형태로 표시되며, X 버튼으로 삭제할 수 있습니다.

### 탐색 태그 (선택)

탐색 페이지에서 캐릭터를 찾을 때 사용되는 태그입니다. 성격 키워드와는 별도로 입력합니다.

| 항목          | 내용  |
| ----------- | --- |
| 최대 개수       | 5개  |
| 태그당 최대 글자 수 | 30자 |

- 예시: `BL`, `HL`, `순애물`

## 4. 상세 프로필

캐릭터의 배경 스토리와 상세 정보를 작성하는 단계입니다. 멀티챗에서는 간소화된 버전이 표시됩니다.

### 기본 정보 (1:1 채팅 전용, 선택)

멀티챗에서는 이 항목이 표시되지 않습니다.

| 항목 | 예시      |
| --- | ------- |
| 나이 | 32세     |
| 키  | 188cm   |
| 생일 | 10월 27일 |

### 좋아하는 것 / 싫어하는 것 (1:1 채팅 전용, 선택)

멀티챗에서는 이 항목이 표시되지 않습니다.

태그 형태로 입력합니다. 입력 후 Enter 키를 눌러 추가합니다.

- **좋아하는 것**: 초록색 태그로 표시
- **싫어하는 것**: 빨간색 태그로 표시

### 상세 소개글 (권장)

캐릭터 또는 멀티챗 콘텐츠의 상세 소개 텍스트를 작성합니다. 이 내용은 대화 시작 전 사용자에게 표시됩니다.

| 콘텐츠 유형 | 라벨     |
| ------ | ------ |
| 1:1 채팅 | 상세 프로필 |
| 멀티챗    | 상세 페이지 |

| 항목      | 내용                                                                 |
| ------- | ------------------------------------------------------------------ |
| 최대 글자 수 | 20,000자                                                            |
| HTML 사용 | 가능                                                                 |
| 지원 태그   | `h1`~`h6`, `p`, `br`, `ul`, `ol`, `li`, `strong`, `em`, `a`, `img` |

"미리보기" 버튼으로 작성한 HTML이 어떻게 보이는지 확인할 수 있습니다.

**작성 예시:**

```html
<h2>기본 정보</h2>
<p>제국의 제3황자로, 냉정하고 오만한 성격을 지녔다.</p>

<h2>성격</h2>
<ul>
  <li>겉으로는 차갑지만 속으로는 따뜻한 면이 있다</li>
  <li>자존심이 강하여 먼저 마음을 표현하지 않는다</li>
</ul>
```

## 5. 인트로씬

대화가 시작될 때 처음 표시되는 **고정 텍스트**를 작성하는 단계입니다. AI가 생성하는 것이 아니라, 크리에이터가 미리 작성해 둔 텍스트가 그대로 표시됩니다.

### 기본 인트로 (필수)

| 항목      | 제한                  |
| ------- | ------------------- |
| 인트로씬 제목 | 최대 100자 (예: "첫 만남") |
| 인트로 내용  | 최소 1자, 최대 2,000자    |

**작성 팁:**

- 캐릭터가 처음 등장하는 장면을 생생하게 묘사하세요.
- 현재 상황, 시간, 장소 등을 구체적으로 설명하면 좋습니다.
- 캐릭터의 첫 인상이나 분위기를 전달할 수 있습니다.
- 멀티챗의 경우, 여러 캐릭터가 등장하는 상황을 자연스럽게 묘사하세요.

### 시점 관리 (선택)

하나의 캐릭터에 여러 시작 상황(시점)을 추가할 수 있습니다. 사용자는 대화를 시작할 때 원하는 시점을 선택할 수 있습니다.

시점별로 다음 정보를 입력합니다.

| 항목       | 제한                      |
| -------- | ----------------------- |
| 시점 이름    | 최대 50자 (예: "방과 후 교실에서") |
| 인트로씬 텍스트 | 최대 2,000자               |

시점 이름은 인트로씬 제목으로도 사용됩니다.

### 엔딩별 인트로 (수정 모드에서만)

캐릭터가 시나리오에 연결되어 있고 시나리오에 엔딩이 설정된 경우, 각 엔딩에 대한 별도의 인트로씬을 설정할 수 있습니다. 이 항목은 캐릭터 수정 시에만 표시됩니다.

## 6. 콘텐츠 설정 (멀티챗 전용)

멀티챗의 전체 대화 맥락을 설정하는 단계입니다. 이 단계는 **멀티챗 유형에서만 표시**되며, 1:1 채팅에서는 표시되지 않습니다.

### 콘텐츠 프롬프트 (권장)

참여 캐릭터 전체가 공유하는 **배경 설정, 규칙, 유저 역할** 등을 작성합니다. 이 내용은 AI에게 시스템 프롬프트로 전달되어, 모든 캐릭터의 대화에 일관된 맥락을 제공합니다.

다음 항목들을 포함하여 작성하는 것을 권장합니다.

1. **배경 상황** - 어떤 세계, 어떤 장소, 어떤 시점인지
2. **캐릭터 간 관계** - 참여 캐릭터들이 서로 어떤 관계인지
3. **유저의 역할** - 유저가 이 대화에서 어떤 입장인지
4. **대화 규칙** - 특별한 설정이나 제한 사항

| 항목    | 내용           |
| ----- | ------------ |
| 최대 토큰 | **1,500 토큰** |

> **참고:** 한글은 글자당 약 1.5토큰, 영문은 글자당 약 0.25토큰으로 계산됩니다. 입력란 상단의 토큰 사용량 표시에서 현재 사용량을 실시간으로 확인할 수 있습니다.

**작성 예시:**

```text
# 배경 상황
제국 황실의 야회가 열리고 있다. 대연회장에는 귀족들이 모여 있다.

# 캐릭터 간 관계
- 제3황자와 근위기사는 어린 시절부터 함께 자란 사이
- 궁정마법사는 황자의 비밀 임무를 돕고 있다

# 유저의 역할
유저는 초대받은 외국 사절단의 일원이다.

# 대화 규칙
- 각 캐릭터는 자신의 신분에 맞는 언어를 사용한다
- 공식적인 자리이므로 격식을 갖춘 대화가 기본이다
```

### 성인 모드 지원 (선택)

본인 인증을 완료한 크리에이터에게만 표시되는 설정입니다.

| 항목          | 내용                               |
| ----------- | -------------------------------- |
| 성인 모드 지원 토글 | 이 멀티챗 콘텐츠가 성인 모드 대화를 지원할지 설정합니다. |
| 성인 프롬프트     | 성인 모드 활성화 시 AI에게 전달되는 추가 지침입니다.  |

## 7. 참여 캐릭터 (멀티챗 전용)

멀티챗에 참여하는 개별 캐릭터를 등록하고 관리하는 단계입니다. 이 단계는 **멀티챗 유형에서만 표시**되며, 1:1 채팅에서는 표시되지 않습니다.

### 캐릭터 수 제한

| 항목    | 내용     |
| ----- | ------ |
| 최소 인원 | **2명** |
| 최대 인원 | **5명** |

최소 2명 이상의 캐릭터를 등록해야 이 단계가 완료됩니다.

### 캐릭터별 설정 항목

각 참여 캐릭터에 대해 다음 정보를 입력합니다.

| 항목      | 필수 여부 | 제한           | 설명                      |
| ------- | ----- | ------------ | ----------------------- |
| 이름      | 필수    | 최대 50자       | 대화에서 표시되는 캐릭터 이름        |
| 프로필 이미지 | 선택    | 이미지 규격 참고    | 대화에서 캐릭터 아바타로 표시        |
| 페르소나    | 필수    | **1,000 토큰** | 캐릭터의 성격, 말투, 비밀 등 AI 지침 |
| 스프라이트   | 선택    | -            | 감정/상황별 일러스트             |

> **참고:** 같은 멀티챗 안에서 캐릭터 이름이 중복될 수 없습니다.

### 페르소나 작성

각 캐릭터의 페르소나에는 다음 내용을 포함하는 것을 권장합니다.

- **성격** - 캐릭터의 핵심 성격 특성
- **말투** - 존댓말/반말, 특정 어미, 대사 예시
- **외형** - 간단한 외형 묘사
- **비밀** - 다른 캐릭터가 모르는 정보 (선택)

| 항목    | 내용                |
| ----- | ----------------- |
| 최대 토큰 | 캐릭터당 **1,000 토큰** |

**작성 예시:**

```text
# 성격
차분하고 이성적이지만, 가끔 장난기가 넘친다.
자존심이 강하지만 동료를 진심으로 아끼는 타입.

# 말투
- 기본적으로 반말을 사용한다
- "......" 으로 말을 흐리는 습관이 있다
- 예시: "......관심 없다고 했을 텐데."

# 외형
은발에 붉은 눈동자. 군복 차림.
```

### 멀티챗 토큰 예산 요약

| 항목             | 토큰 한도      |
| -------------- | ---------- |
| 콘텐츠 프롬프트 (6단계) | 1,500 토큰   |
| 캐릭터 페르소나 (개별)  | 각 1,000 토큰 |
| 최대 캐릭터 수       | 5명         |

## 8. AI 설정 (1:1 채팅 전용)

AI가 캐릭터를 연기할 때 참고하는 **페르소나 프롬프트**를 작성하는 단계입니다. 이 내용이 충실할수록 캐릭터의 대화 품질이 높아집니다.

> **참고:** 이 단계는 **1:1 채팅 유형에서만 표시**됩니다. 멀티챗에서는 [콘텐츠 설정](#6-콘텐츠-설정-멀티챗-전용)과 [참여 캐릭터](#7-참여-캐릭터-멀티챗-전용)의 페르소나로 대체됩니다.

### 토큰 예산 시스템

캐릭터의 AI 설정은 글자 수가 아닌 **토큰 예산**으로 관리됩니다. 전체 페르소나 프롬프트, 문체 묘사, 호감도 단계별 말투 프롬프트의 토큰 합계가 예산 내에 있어야 합니다.

| 항목          | 내용                          |
| ----------- | --------------------------- |
| 기본 토큰 예산    | **4,000 토큰**                |
| 성인 모드 활성화 시 | **\+1,000 토큰** (총 5,000 토큰) |

> **참고:** 한글은 글자당 약 1.5토큰, 영문은 글자당 약 0.25토큰으로 계산됩니다. 상단의 토큰 사용량 바에서 현재 사용량을 실시간으로 확인할 수 있습니다.

### 전체 페르소나 프롬프트 (권장)

다음 항목들을 포함하여 작성하는 것을 권장합니다.

1. **외형 묘사** - 머리 색, 눈 색, 체형, 복장 등
2. **역할/신분** - 직업, 신분, 사회적 지위 등
3. **성격 및 특징** - 성격, 습관, 특이사항 등
4. **말투 규칙** - 존댓말/반말, 특정 어미 사용 등
5. **말투 예시** - 실제 대사 예시 3~5개

**작성 예시:**

```text
# 외형
은발에 붉은 눈동자. 제국 황실의 군복을 입고 있다.
키가 크고 날카로운 인상이지만, 웃을 때는 부드러운 인상으로 바뀐다.

# 역할/신분
제국의 제3황자. 군부 총사령관을 겸직하고 있다.

# 성격 및 특징
- 겉으로는 냉정하고 무심하지만, 진심으로 아끼는 사람에겐 다정하다
- 감정 표현이 서투르다
- 홍차를 즐겨 마신다

# 말투 규칙
- 기본적으로 반말을 사용한다
- 공식적인 자리에서는 격식체를 사용한다
- "......"으로 말끝을 흐리는 습관이 있다

# 말투 예시
- "......관심 없다고 했을 텐데."
- "네가 다칠까 봐 그런 거지, 다른 의미는 없어."
- "......한 번만. 한 번만 더 그렇게 웃어 봐."
```

### 문체 묘사 (선택)

캐릭터의 서술 문체, 톤, 분위기를 지정합니다. 비워 두면 루벳 기본 문체 설정이 적용됩니다. 토큰 예산에 포함되므로 상단 토큰 사용량 바를 확인하며 작성하세요.

### 성인 모드 지원 (선택)

본인 인증을 완료한 크리에이터에게만 표시되는 설정입니다.

| 항목          | 내용                              |
| ----------- | ------------------------------- |
| 성인 모드 지원 토글 | 이 캐릭터가 성인 모드 대화를 지원할지 설정합니다.    |
| 성인 프롬프트     | 성인 모드 활성화 시 AI에게 전달되는 추가 지침입니다. |

- 성인 모드를 활성화하면 토큰 예산이 **\+1,000 토큰** 추가됩니다.
- 사용자가 채팅방을 생성할 때, 해당 캐릭터가 성인 모드를 지원하고 사용자도 본인 인증을 완료한 경우에만 성인 모드 채팅방을 만들 수 있습니다.

## 9. 스프라이트

대화 중 상황에 맞게 표시되는 캐릭터 일러스트를 관리하는 단계입니다.

### 스프라이트란?

스프라이트는 대화 중 캐릭터의 감정이나 상황에 따라 바뀌는 일러스트입니다. AI가 대화 내용을 분석하여 상황에 맞는 스프라이트를 자동으로 표시합니다.

### 기본 슬롯

처음에 7개의 기본 슬롯이 제공됩니다.

| 슬롯  | 이름  | 무료/유료      | 기본 키워드                |
| --- | --- | ---------- | --------------------- |
| 기본  | 기본  | 항상 무료      | (없음)                  |
| 행복  | 행복  | 유료 (100하트) | 기쁨, 행복, 웃음, 미소        |
| 슬픔  | 슬픔  | 유료 (100하트) | 슬픔, 눈물, 우울, 서글픔       |
| 분노  | 분노  | 유료 (100하트) | 화남, 분노, 짜증, 열받음       |
| 놀람  | 놀람  | 유료 (100하트) | 놀람, 깜짝, 충격, 당황        |
| 수줍음 | 수줍음 | 유료 (100하트) | 부끄러움, 수줍음, 창피, 얼굴 빨개짐 |
| 생각  | 생각  | 유료 (100하트) | 고민, 생각, 걱정, 망설임       |

### 슬롯 추가 및 관리

- "스프라이트 추가" 버튼으로 새 슬롯을 추가할 수 있습니다.
- **최대 20개**까지 슬롯을 추가할 수 있습니다.
- 기본 슬롯을 제외한 모든 슬롯은 유료(기본 100하트)로 생성됩니다.

### 슬롯별 설정 항목

| 항목       | 설명                                                     |
| -------- | ------------------------------------------------------ |
| 이미지      | 클릭하여 이미지 또는 영상 업로드 (정사각형 비율)                           |
| 스프라이트 이름 | 슬롯 이름을 자유롭게 변경 가능                                      |
| 상황 키워드   | AI가 대화 상황을 분석해 매칭하는 키워드 (키워드당 최대 20자, Enter 또는 쉼표로 추가) |
| 가격       | 유료 슬롯의 가격 설정 (단위: 하트, 기본값 100)                         |

### 키워드 해금 방식

채팅 중 키워드가 매칭되면 해당 스프라이트가 **영구 해금**됩니다. 해금된 스프라이트는 일러스트 앨범에서도 확인할 수 있습니다.

### 멀티챗의 스프라이트

멀티챗에서는 각 참여 캐릭터가 개별적으로 스프라이트를 가질 수 있습니다. [참여 캐릭터](#7-참여-캐릭터-멀티챗-전용) 단계에서 캐릭터별 스프라이트를 설정합니다.

### 수익

유료 스프라이트가 판매되면 설정한 가격의 \*\*50%\*\*를 수익으로 받습니다. 자세한 내용은 [수익 창출 가이드](/monetization-guide)를 참고하세요.

### 팁

- "기본" 슬롯은 항상 무료이며 변경할 수 없습니다.
- 이미지가 첨부되지 않은 슬롯은 대화 중 표시되지 않습니다.
- 특별 의상, 레어 표정, 한정판 일러스트 등에 높은 가격을 설정해 보세요.
- 영상 스프라이트의 기본 가격은 500하트입니다.

## 10. 호감도 설정 (1:1 채팅 전용)

호감도에 따라 캐릭터의 호칭과 말투가 변화하도록 설정하는 단계입니다.

> **참고:** 이 단계는 **1:1 채팅 유형에서만 표시**됩니다. 멀티챗에서는 호감도 시스템이 적용되지 않습니다.

### 호감도 단계란?

사용자와의 대화를 통해 캐릭터의 호감도가 변화하며, 호감도 수치에 따라 캐릭터가 사용자를 부르는 호칭이나 말투가 달라집니다.

### 단계 추가

"+ 단계 추가" 버튼으로 새 호감도 단계를 추가합니다.

### 단계별 설정 항목

| 항목      | 설명                            | 제한      |
| ------- | ----------------------------- | ------- |
| 기준 수치   | 이 호감도 이상일 때 해당 단계가 적용됩니다      | 숫자      |
| 단계명     | 단계의 이름입니다 (예: 남남, 지인, 친구, 연인) | 최대 100자 |
| 호칭      | 캐릭터가 사용자를 부르는 호칭입니다           | 최대 100자 |
| 말투 프롬프트 | AI에게 전달되는 이 단계의 행동 지침입니다      | 최대 200자 |

### 사용 가능한 변수

말투 프롬프트에서 다음 변수를 사용할 수 있습니다.

| 변수                  | 의미        |
| ------------------- | --------- |
| `{{userName}}`      | 사용자 이름    |
| `{{characterName}}` | 캐릭터 이름    |
| `{{affection}}`     | 현재 호감도 수치 |
| `{{stageLabel}}`    | 현재 단계명    |
| `{{honorific}}`     | 현재 호칭     |

### 설정 예시

| 기준 수치 | 단계명 | 호칭           | 말투 프롬프트                       |
| ----- | --- | ------------ | ----------------------------- |
| 0     | 남남  | 당신           | 기본적으로 차가운 태도를 유지한다. 경계심을 보인다. |
| 30    | 지인  | 너            | 조금 관심을 보이기 시작한다. 여전히 무심한 척한다. |
| 70    | 연인  | userName(이)야 | 진심으로 다정하게 대한다. 애칭을 사용한다.      |

## 11. 검토 및 생성

모든 설정을 최종 확인하고 캐릭터를 생성하는 단계입니다.

### 완료 현황 확인

각 단계의 완료 여부가 카드 형태로 표시됩니다. 초록색 체크 표시가 있으면 해당 단계가 완료된 상태입니다. 표시되는 카드는 콘텐츠 유형에 따라 다릅니다.

**1:1 채팅:**

| 카드     | 완료 조건                  |
| ------ | ---------------------- |
| 기본 정보  | 이름이 입력됨                |
| 상세 프로필 | 상세 소개글이 작성됨            |
| AI 설정  | 페르소나 프롬프트가 작성됨         |
| 스프라이트  | 1개 이상의 스프라이트 이미지가 업로드됨 |
| 호감도 설정 | 1개 이상의 호감도 단계가 설정됨     |

**멀티챗:**

| 카드     | 완료 조건                              |
| ------ | ---------------------------------- |
| 기본 정보  | 제목이 입력됨                            |
| 콘텐츠 설정 | 콘텐츠 프롬프트가 작성됨                      |
| 참여 캐릭터 | 2명 이상의 캐릭터가 등록되고, 각각 이름과 페르소나가 작성됨 |
| 스프라이트  | 1개 이상의 스프라이트 이미지가 업로드됨             |

각 카드의 "수정" 버튼으로 해당 단계로 바로 이동할 수 있습니다.

### 수익 창출 체크리스트

유료 스프라이트가 1개 이상 설정된 경우, **크리에이터 선언 체크리스트**가 표시됩니다. 3가지 항목 모두에 동의해야 유료 판매가 가능합니다. 체크리스트에 대한 자세한 내용은 [수익 창출 가이드](/monetization-guide)를 참고하세요.

### 공개 설정

두 가지 옵션 중 하나를 선택합니다.

| 옵션      | 설명                                     |
| ------- | -------------------------------------- |
| 비공개     | 나만 채팅할 수 있습니다. 심사 없이 바로 사용 가능합니다.      |
| 공개 (제출) | 다른 사용자도 채팅할 수 있습니다. 관리자 심사 승인 후 공개됩니다. |

### 저장

- **새 캐릭터**: "캐릭터 생성" 버튼을 클릭하면 캐릭터가 생성됩니다.
- **수정 모드 + 공개 제출**: "제출하기" 버튼을 클릭하면 심사에 제출됩니다.
- **수정 모드 + 비공개**: "캐릭터 저장" 버튼을 클릭하면 변경사항이 저장됩니다.

## 13. 임시 저장 기능

새 캐릭터를 만드는 도중 작업 내용이 사라지지 않도록 임시 저장 기능을 제공합니다.

### 사용 방법

1. 상단의 "임시저장" 버튼을 클릭합니다.
2. 현재까지 작성한 모든 내용이 브라우저에 저장됩니다.
3. 다음에 새 캐릭터 만들기 페이지에 접속하면 "이어서 작성하시겠습니까?" 대화상자가 표시됩니다.
4. "이어서 작성"을 선택하면 이전 작업을 이어갈 수 있습니다.

### 주의사항

- 임시 저장은 **새 캐릭터 만들기**에서만 동작합니다 (수정 모드에서는 자동 저장이 서버에 반영됩니다).
- 임시 저장 데이터는 브라우저의 로컬 저장소에 보관되며, 최대 5MB까지 저장 가능합니다.
- 캐릭터 생성이 완료되면 임시 저장 데이터는 자동으로 삭제됩니다.
- 멀티챗 콘텐츠의 임시 저장 시 참여 캐릭터 데이터(이름, 페르소나 등)도 함께 저장됩니다.

## 14. 공개 및 심사 절차

캐릭터는 기본적으로 **바로 공개**가 가능합니다. 별도의 심사 없이 공개 설정만 하면 다른 사용자에게 즉시 노출됩니다.

단, 이식(다른 플랫폼에서 가져온 캐릭터) 등 특정 케이스에서는 관리자 심사가 진행될 수 있습니다.

### 공개 흐름

```text
비공개 (초안)  →  공개 설정  →  즉시 공개
```

### 심사가 진행되는 경우

이식 등 일부 케이스에서는 관리자 심사가 필요합니다.

```text
비공개 (초안)  →  심사 제출  →  심사 대기 중  →  승인됨 / 거절됨
```

### 각 상태별 안내

| 상태       | 설명                     | 가능한 작업              |
| -------- | ---------------------- | ------------------- |
| 비공개 (초안) | 나만 볼 수 있는 상태           | 수정, 삭제, 공개 또는 심사 제출 |
| 공개됨      | 다른 사용자에게 공개된 상태        | 수정 후 저장 가능          |
| 심사 대기 중  | 관리자가 검토하고 있는 상태 (이식 등) | 제출 취소 (비공개로 전환됨)    |
| 승인됨      | 심사를 통과하여 공개된 상태        | 수정 후 저장 가능          |
| 거절됨      | 심사에서 거절된 상태 (거절 사유 표시) | 수정 후 다시 제출 가능       |

### 심사 제출 시 확인 사항 (이식 등 심사 대상인 경우)

- 심사 중에는 **캐릭터 수정이 제한**됩니다.
- 제출을 취소하면 비공개 상태로 전환되며, 수정 후 다시 제출할 수 있습니다.
- 하루 최대 **10건**까지 심사 제출이 가능합니다.

### 심사 기준

심사 시 다음 사항이 검토됩니다.

- 이름: 2자 이상, 50자 이하
- 상세 설명: 10자 이상, 20,000자 이하
- 금지어 포함 여부
- 콘텐츠 규정 준수 여부

## 15. 콘텐츠 규정

캐릭터를 제작할 때 다음 규정을 준수해야 합니다.

### 저작권

- **오리지널 작품**만 수익 창출이 가능합니다.
- 타인의 이미지, 텍스트, 음악 등을 무단으로 사용할 수 없습니다.
- 2차 창작(기존 웹소설, 만화, 게임, 영화 등을 기반으로 한 작품)은 **무료 공개만** 가능하며, 수익 창출이 제한됩니다.

### 콘텐츠 제한

- 성인 콘텐츠는 본인 인증을 완료한 크리에이터만 제작할 수 있으며, 수익 창출도 가능합니다.
- 금지어가 포함된 콘텐츠는 심사에서 거절됩니다.
- 커뮤니티 가이드라인을 위반한 경우 콘텐츠가 삭제되거나 계정이 정지될 수 있습니다.

### 위반 시 제재

- 저작권 침해 소지가 있는 콘텐츠는 수익 창출이 차단됩니다.
- 반복 위반 시 콘텐츠 삭제 또는 계정 정지 조치가 이루어질 수 있습니다.

## 16. 자주 묻는 질문

### Q. 1:1 채팅과 멀티챗의 차이는 무엇인가요?

1:1 채팅은 하나의 캐릭터와 1대1로 대화하는 방식이고, 멀티챗은 2~5명의 캐릭터가 하나의 채팅방에서 함께 대화하는 그룹 채팅 방식입니다. 1:1 채팅에서는 AI 설정과 호감도를 설정하고, 멀티챗에서는 콘텐츠 설정과 참여 캐릭터를 설정합니다.

### Q. 1:1 채팅과 멀티챗 간 유형을 전환할 수 있나요?

제작 중에는 유형을 전환할 수 있습니다. 숨겨진 단계의 데이터는 메모리에 보존되므로 다시 전환하면 복원됩니다. 단, 이미 생성/저장된 캐릭터의 유형 전환은 권장하지 않습니다.

### Q. 멀티챗 참여 캐릭터 수 제한은 어떻게 되나요?

최소 2명, 최대 5명까지 등록할 수 있습니다.

### Q. 캐릭터를 꼭 시나리오에 연결해야 하나요?

아닙니다. 캐릭터는 독립적으로 존재할 수 있으며, 시나리오 없이도 "자유 대화" 모드로 채팅할 수 있습니다. 시나리오에 연결하면 스토리 맥락 안에서 더 풍부한 대화가 가능합니다.

### Q. 멀티챗 콘텐츠를 시나리오에 연결할 수 있나요?

현재 멀티챗 콘텐츠는 시나리오와 연동할 수 없습니다. 시나리오에는 1:1 채팅 캐릭터만 등록할 수 있습니다.

### Q. 비공개 상태에서도 채팅이 가능한가요?

가능합니다. 비공개 캐릭터는 크리에이터 본인만 채팅할 수 있습니다. 테스트 용도로 활용하시면 됩니다.

### Q. 승인된 캐릭터를 수정할 수 있나요?

가능합니다. 승인된 캐릭터도 수정 후 저장할 수 있습니다.

### Q. 거절된 캐릭터는 어떻게 하나요?

거절 사유를 확인하고, 해당 부분을 수정한 후 다시 심사에 제출할 수 있습니다.

### Q. 캐릭터를 삭제할 수 있나요?

비공개, 심사 대기, 거절 상태의 캐릭터는 삭제할 수 있습니다.

### Q. AI 설정(페르소나)을 비워두면 어떻게 되나요?

기본적인 대화는 가능하지만, 캐릭터만의 고유한 성격이나 말투가 반영되지 않아 대화 품질이 떨어질 수 있습니다. 가능한 한 상세하게 작성하는 것을 권장합니다.

### Q. 스프라이트를 꼭 설정해야 하나요?

필수는 아닙니다. 스프라이트가 없어도 대화는 정상적으로 진행됩니다. 다만, 스프라이트가 있으면 시각적으로 더 풍부한 대화 경험을 제공하며, 수익 창출 수단으로도 활용할 수 있습니다.
