## 시작하기 전에: 준비물

시나리오 제작을 시작하기 전에, 아래 항목을 미리 준비해두면 작업이 훨씬 수월합니다.

### 필요한 환경

- **데스크탑 PC 또는 노트북** — 시나리오 에디터는 데스크탑 전용입니다. 모바일에서는 편집할 수 없습니다.
- **권장 브라우저** — Chrome, Edge 등 최신 Chromium 기반 브라우저를 권장합니다.
- **루벳 계정** — 회원가입 및 온보딩이 완료된 상태여야 합니다.

### 미리 준비하면 좋은 이미지 에셋

시나리오에 쓰이는 이미지는 크게 4종류입니다. 아래 표를 보고 필요한 이미지를 미리 준비해두세요.

| 에셋 종류          | 용도                   | 비율/형태        | 포맷              | 준비 팁                              |
| -------------- | -------------------- | ------------ | --------------- | --------------------------------- |
| **썸네일**        | 시나리오 목록에 표시되는 대표 이미지 | **3:4** 세로형  | JPEG, PNG, WebP | 고화질 세로 이미지를 준비하세요                 |
| **스프라이트**      | 대사 장면에 표시되는 캐릭터 일러스트 | **1:1** 정사각형 | **PNG** (투명 배경) | 배경이 없는 누끼 이미지가 필요합니다. 캐릭터당 최대 20장 |
| **캐릭터 대표 이미지** | 캐릭터 프로필에 표시          | **3:4** 세로형  | JPEG, PNG, WebP | 썸네일과 동일한 비율입니다                    |
| **배경 이미지**     | 장면의 배경               | **가로형** 권장   | JPEG, PNG, WebP | 가로로 긴 이미지를 준비하세요                  |

> **용량 안내:** 이미지는 최대 5MB까지 업로드할 수 있으며, 자동으로 2MB 이하로 압축됩니다 (최대 1920px로 리사이즈). 별도로 리사이즈할 필요는 없습니다.

> **BGM은 YouTube 링크만 지원합니다.** 오디오 파일 업로드는 지원하지 않으니, 사용할 BGM의 YouTube 링크를 미리 찾아두세요.

상세 스펙은 이미지/에셋 상세 스펙 섹션에서 확인할 수 있습니다.

{/*  IMAGE: assets-spec-summary — 에셋 종류별 비율/포맷을 한눈에 비교하는 인포그래픽 (썸네일 3:4, 스프라이트 1:1 누끼, 배경 가로형 등)  */}

### 미리 준비하면 좋은 텍스트

- **시나리오 제목** (3~100자)
- **시나리오 설명** (10~500자) — 독자에게 보여지는 소개글입니다.
- **대사 초안** — 메모장이나 문서에 미리 써두면, 나중에 대사 일괄 가져오기 기능으로 한 번에 입력할 수 있습니다.

## 시나리오란?

루벳 시나리오는 **비주얼 노벨** 형식의 인터랙티브 스토리입니다. 독자는 스토리를 읽으며 선택지를 고르고, 선택에 따라 달라지는 전개와 다양한 엔딩을 경험합니다.

시나리오는 **노드(node)** 라는 장면 단위를 만들고, 노드끼리 연결하는 방식으로 구성합니다. 쉽게 말해 **플로우차트**를 그린다고 생각하면 됩니다.

```text
[시작] → [대사1] → [선택지] → [선택A 루트] → [엔딩A]
                             → [선택B 루트] → [엔딩B]
```

{/*  IMAGE: scenario-play-screen — 완성된 시나리오를 플레이하는 화면 (배경 + 캐릭터 스프라이트 + 대사 말풍선 + 선택지가 보이는 상태)  */}

## Step 1. 새 시나리오 만들기

### 1-1. 제작 스튜디오로 이동합니다

루벳에 로그인한 상태에서 **제작 스튜디오** 페이지로 이동합니다.

{/*  IMAGE: studio-dashboard — 스튜디오 메인 화면. "새 시나리오" 버튼 위치를 화살표로 표시  */}

### 1-2. "새 시나리오" 버튼을 클릭합니다

"내 시나리오" 섹션에서 **새 시나리오** 버튼을 클릭하면, 자동으로 제목이 생성되며 (예: "새 시나리오 1") 에디터 화면으로 이동합니다.

### 1-3. 에디터 화면을 둘러봅니다

에디터에 처음 들어오면 **인터랙티브 투어**가 자동으로 시작됩니다. 투어에서 에디터의 주요 영역을 하나씩 안내하니, 끝까지 따라가보세요.

나중에 다시 보고 싶으면 에디터 우측 상단의 **도움말(?) 버튼**을 클릭하면 됩니다.

{/*  IMAGE: editor-initial — 에디터 첫 진입 화면. 빈 캔버스에 "시작" 노드 1개가 있는 상태  */}

{/*  IMAGE: editor-tour — 에디터 투어가 실행되어 특정 영역을 하이라이트하고 설명하는 화면  */}

에디터는 크게 세 영역으로 구성되어 있습니다.

| 영역       | 위치    | 역할                              |
| -------- | ----- | ------------------------------- |
| **캔버스**  | 화면 중앙 | 노드를 배치하고 연결하는 작업 공간             |
| **툴바**   | 화면 상단 | 노드 추가, 캐릭터 관리, 설정, 미리보기 등 주요 도구 |
| **인스펙터** | 화면 우측 | 선택한 노드의 대사, 배경, 연출 등을 편집하는 패널   |

> **이 단계의 결과:** 빈 캔버스에 "시작" 노드가 1개 있는 에디터 화면이 열려 있습니다.

## Step 2. 기본 정보 채우기

시나리오의 제목, 설명, 썸네일 등 기본 정보를 설정합니다.

### 2-1. 설정 패널을 엽니다

툴바에서 **설정(톱니바퀴) 버튼**을 클릭하면 설정 패널이 열립니다.

{/*  IMAGE: settings-basic-info — 설정 패널의 "기본 정보" 탭. 제목, 설명, 태그, 썸네일 입력란이 보이는 상태  */}

### 2-2. 기본 정보를 입력합니다

| 항목           | 설명                             | 제한         |
| ------------ | ------------------------------ | ---------- |
| **제목**       | 시나리오 이름입니다. 독자가 가장 먼저 보게 됩니다.  | 3~100자     |
| **상세 설명**    | 시나리오의 소개글입니다.                  | 10~500자    |
| **태그**       | 검색과 분류에 사용됩니다.                 | 최대 5개      |
| **썸네일 이미지**  | 시나리오 목록에 표시되는 대표 이미지입니다.       | 3:4 세로형 권장 |
| **예상 플레이타임** | 시나리오를 플레이하는 데 걸리는 예상 시간(분)입니다. | 선택 사항      |

> **Tip:** 제목과 설명은 나중에 언제든 수정할 수 있습니다. 우선 임시로 채우고 넘어가도 괜찮습니다.

> **이 단계의 결과:** 설정에 제목, 설명, 썸네일이 채워진 상태입니다.

## Step 3. 캐릭터 배치하기

시나리오에 등장하는 캐릭터를 등록합니다. 툴바에서 **캐릭터 관리** 버튼을 클릭하면 캐릭터 관리 패널이 열립니다.

{/*  IMAGE: character-manager — 캐릭터 관리 패널. 프리셋(주인공), 커스텀 캐릭터(공략캐), 조연·엑스트라 빠른 추가, 하단에 "라이브러리에서 추가"와 "+ 새 캐릭터" 버튼이 보이는 상태  */}

캐릭터 관리 패널은 세 영역으로 나뉩니다.

### 주인공 (프리셋 · 고정)

주인공 캐릭터는 항상 고정으로 존재합니다. **이름만 변경**할 수 있습니다. (이는 구분을 쉽게 하기위함이며 기본적으로 `{{user}}` 변수로 플레이하는 사용자의 이름이 표시됩니다.)

### 공략 캐릭터 (커스텀 캐릭터)

공략 캐릭터는 **라이브러리에서 가져오는 방식**으로만 추가할 수 있습니다.

1. **이미 만든 캐릭터가 있는 경우** — 하단의 **"라이브러리에서 추가"** 버튼을 클릭하여, 스튜디오에서 미리 만들어 둔 캐릭터 중에서 선택합니다.
2. **새로 만들어야 하는 경우** — 하단의 **"+ 새 캐릭터"** 버튼을 클릭하면 스튜디오의 캐릭터 생성 페이지로 이동합니다. 캐릭터를 만든 뒤, 다시 에디터로 돌아와서 라이브러리에서 가져오면 됩니다.

공략 캐릭터의 상세 설정(대표 이미지, 스프라이트, AI 페르소나 등)은 에디터가 아니라 **스튜디오 캐릭터 페이지**에서 편집합니다. 캐릭터 옆의 **외부 링크 아이콘**을 클릭하면 해당 캐릭터의 편집 페이지로 이동합니다.

> **캐릭터 만드는 방법이 궁금하다면?** → [캐릭터 작성 가이드](/character-creation-guide)에서 AI 페르소나, 인트로씬, 스프라이트 등록 등 자세한 내용을 확인할 수 있습니다.

### 조연 · 엑스트라 (빠른 추가)

대사에만 잠깐 등장하는 조연이나 엑스트라는 **이름만 입력**하면 바로 추가할 수 있습니다. "조연 · 엑스트라 빠른 추가" 입력란에 이름을 쓰고 **추가** 버튼을 클릭하세요. (예: "친구 A", "지나가는 행인")

### 캐릭터 유형 정리

| 유형            | 추가 방법                      | 상세 설정             | 설명                                |
| ------------- | -------------------------- | ----------------- | --------------------------------- |
| **주인공**       | 프리셋 (자동 생성)                | 이름만 변경 가능         | 플레이어의 분신. 항상 고정으로 존재합니다.          |
| **공략 캐릭터**    | 라이브러리에서 추가 / 새 캐릭터 생성 후 추가 | 스튜디오 캐릭터 페이지에서 편집 | AI 채팅, 스프라이트, 페르소나 등 풀 설정이 가능합니다. |
| **조연 · 엑스트라** | 이름 입력으로 빠른 추가              | 이름만 설정            | 대사에만 등장하는 간단한 캐릭터입니다.             |

> **참고:** 현재 시나리오에는 **1:1 채팅 캐릭터만** 공략 캐릭터로 등록할 수 있습니다. 멀티챗 콘텐츠는 시나리오와 연동할 수 없습니다.

> **이 단계의 결과:** 시나리오에 캐릭터가 1명 이상 등록된 상태입니다.

## Step 4. 대사 작성하기

이제 실제 스토리를 쓸 차례입니다. 노드를 만들고, 그 안에 대사를 채워넣습니다.

### 4-1. 노드를 추가합니다

툴바의 **\+ 버튼**을 클릭하면 캔버스에 새 노드가 추가됩니다.

{/*  IMAGE: node-add — 툴바의 + 버튼 위치를 화살표로 표시하고, 캔버스에 새 노드가 추가된 상태  */}

### 4-2. 대사를 편집합니다

캔버스에서 노드를 클릭하면 우측에 **인스펙터** 패널이 열립니다. 인스펙터에서 대사를 작성하고 편집합니다.

{/*  IMAGE: inspector-dialogue — 인스펙터에서 대사를 편집하는 화면. 캐릭터 선택, 메시지 유형, 텍스트 입력란이 보이는 상태  */}

하나의 노드 안에 **여러 줄의 대사**를 순서대로 넣을 수 있습니다. "대사 추가" 버튼을 누르면 줄이 추가됩니다.

각 대사 줄에서 설정할 수 있는 항목:

| 항목         | 설명                                        |
| ---------- | ----------------------------------------- |
| **캐릭터**    | 이 대사를 말하는 캐릭터를 선택합니다. 선택하지 않으면 나레이션이 됩니다. |
| **메시지 유형** | 대사(말풍선) / 나레이션(전체 텍스트) / 연출(장면 전환 등)      |
| **텍스트**    | 실제 대사 또는 나레이션 내용                          |
| **속마음**    | 캐릭터의 내면 묘사 (이탤릭체로 표시됨)                    |
| **스프라이트**  | 이 대사에서 표시할 캐릭터 일러스트                       |
| **효과음**    | 이 대사와 함께 재생할 효과음                          |

대사 줄은 드래그로 **순서 변경**, 화살표 버튼으로 **위/아래 이동**, 특정 줄에서 **노드 분할** 등이 가능합니다.

### 템플릿 변수: `{{user}}`

대사 텍스트 입력란 아래에  **`{{user}}`** 버튼이 표시됩니다. 이 버튼을 클릭하면 커서 위치에 `{{user}}`가 삽입되며, 플레이 시 **독자(플레이어)의 이름**으로 자동 치환됩니다.

캐릭터가 독자의 이름을 부르는 대사를 작성할 때 사용합니다.

**예시:**

```text
{{user}}씨, 오늘 기분이 어때요?
→ (플레이 시) 루벳씨, 오늘 기분이 어때요?
```

> **Tip:** 버튼 옆의 도움말 아이콘(?)을 클릭하면 설명을 볼 수 있습니다.

### 4-3. 시작 노드를 지정합니다

시나리오의 **첫 번째 노드**를 시작 노드로 지정해야 합니다. 인스펙터에서 "시작 노드로 지정" 옵션을 설정할 수 있습니다.

> **Tip:** 새 시나리오를 만들면 기본으로 "시작" 노드가 하나 생성되어 있습니다. 이 노드가 시작점 역할을 합니다.

### 대사를 한 번에 붙여넣기: 대사 일괄 가져오기

대사를 미리 메모장 등에 써둔 경우, 툴바의 **대사 일괄 가져오기** 기능을 사용하면 한 번에 여러 대사를 입력할 수 있습니다.

{/*  IMAGE: bulk-import-modal — 대사 일괄 가져오기 모달. 텍스트 입력란과 미리보기가 보이는 상태  */}

**형식은 간단합니다:**

```text
그 말에 가슴이 서늘해졌다.
카일: 예상했어. 그 자식도 뭔가 느꼈겠지.
나: 마주치면 어쩌죠?
시온: 막으면 베면 돼.
```

| 규칙                    | 설명                     |
| --------------------- | ---------------------- |
| `캐릭터이름: 대사`           | 해당 캐릭터의 대사로 인식됩니다      |
| 콜론 없는 줄               | **나레이션(지문)** 으로 인식됩니다  |
| `나:`, `주인공:`, `플레이어:` | 주인공(me) 캐릭터의 대사로 인식됩니다 |

캐릭터 이름은 시나리오에 등록된 캐릭터와 자동으로 매칭됩니다. 매칭되지 않는 화자는 경고 표시가 뜨며, 가져온 후에 수정할 수 있습니다.

가져오기 모드는 두 가지입니다:

- **이어붙이기** — 기존 대사 뒤에 추가
- **덮어쓰기** — 기존 대사를 교체 (확인 팝업이 뜹니다)

> **이 단계의 결과:** 노드 안에 대사가 채워져 있고, 시작 노드가 지정된 상태입니다.

## Step 5. 노드 연결하기

노드를 만들었으면, 노드끼리 연결해서 스토리의 **흐름**을 만듭니다.

### 5-1. 노드를 연결합니다

1. 노드 하단의 **출력 핸들(동그라미)** 에 마우스를 올립니다.
2. 드래그하여 다른 노드 상단의 **입력 핸들**에 연결합니다.

이것만으로 "이 노드 다음에 저 노드로 넘어간다"는 흐름이 만들어집니다.

{/*  IMAGE: node-connect — 노드의 출력 핸들에서 다른 노드의 입력 핸들로 드래그하여 연결하는 과정  */}

### 5-2. 플로우 구조를 이해합니다

{/*  IMAGE: flow-example — 3~5개 노드가 연결된 간단한 플로우차트 완성 예시  */}

시나리오의 흐름은 크게 세 가지 구조로 만들 수 있습니다.

**일직선 구조** — 가장 기본적인 형태입니다:

```text
시작 → 대사1 → 대사2 → 대사3 → 엔딩
```

**분기 구조** — 선택지에 따라 다른 경로로 나뉩니다:

```text
시작 → 선택지 → [선택A] → 루트A → 엔딩A
               → [선택B] → 루트B → 엔딩B
```

**합류 구조** — 나뉘었던 경로가 다시 하나로 합쳐집니다:

```text
시작 → 선택지 → [선택A] → 합류 지점 → 공통 진행 → 엔딩
               → [선택B] ↗
```

> **Tip:** 처음에는 일직선 구조로 시작하고, 익숙해지면 분기/합류를 추가하는 것을 추천합니다.

> **이 단계의 결과:** 여러 노드가 화살표로 연결되어 스토리 흐름이 만들어진 상태입니다.

## Step 6. 선택지 만들기

선택지는 독자가 스토리의 방향을 결정하는 핵심 요소입니다.

### 6-1. 선택지를 추가합니다

1. 캔버스에서 선택지를 넣을 노드를 클릭합니다.
2. 인스펙터의 **선택지** 섹션에서 **선택지 추가** 버튼을 클릭합니다.
3. 각 선택지의 텍스트를 입력합니다.

{/*  IMAGE: choice-editor — 인스펙터의 선택지 편집 UI. 선택지 텍스트 입력란과 연결 노드 설정이 보이는 상태  */}

### 6-2. 선택지를 다른 노드에 연결합니다

선택지가 있는 노드는 각 선택지마다 **별도의 연결**을 설정합니다. 캔버스에서 각 선택지의 출력 핸들을 다른 노드로 드래그하여 연결합니다.

{/*  IMAGE: branch-flow — 선택지 2개가 각각 다른 노드로 연결된 분기 구조 완성 예시  */}

### 선택지 설정 항목

| 항목         | 설명                                        |
| ---------- | ----------------------------------------- |
| **텍스트**    | 독자에게 표시되는 선택지 문구                          |
| **다음 노드**  | 이 선택지를 고르면 이동할 노드                         |
| **호감도 변화** | 선택 시 캐릭터별 호감도 증감 값. 양수(+)는 증가, 음수(-)는 감소. |

> **이 단계의 결과:** 선택지가 있는 노드가 만들어졌고, 각 선택지가 다른 노드로 연결된 상태입니다.

## Step 7. 배경과 연출 넣기

대사와 선택지만으로도 시나리오는 동작하지만, 배경 이미지와 연출을 추가하면 훨씬 몰입감이 올라갑니다.

### 배경 이미지

캔버스에서 노드를 선택하고, 인스펙터에서 **배경** 섹션을 찾습니다.

| 항목         | 옵션                        | 설명                                                              |
| ---------- | ------------------------- | --------------------------------------------------------------- |
| **모드**     | 상속 / 설정 / 숨기기             | **상속**: 이전 노드의 배경을 그대로 사용. **설정**: 새 배경 이미지 지정. **숨기기**: 배경 없음. |
| **배경 이미지** | 에셋 탐색기에서 선택               | 배경으로 사용할 이미지를 선택하거나 업로드합니다                                      |
| **시간 필터**  | 없음 / 낮 / 석양 / 밤 / 새벽 / 흐림 | 같은 배경 이미지에 시간대별 색감 필터를 씌울 수 있습니다                                |

> **Tip:** 배경 모드를 "상속"으로 두면, 이전 노드에서 설정한 배경이 자동으로 유지됩니다. 장면이 바뀌는 노드에서만 "설정"으로 변경하면 됩니다.

{/*  IMAGE: background-setting — 인스펙터의 배경/연출 설정 UI. 배경 모드 선택, 이미지 선택, 시간 필터 등이 보이는 상태  */}

### 스프라이트 (캐릭터 일러스트)

| 항목     | 옵션            | 설명                                    |
| ------ | ------------- | ------------------------------------- |
| **모드** | 상속 / 설정 / 숨기기 | 배경과 동일한 방식으로 작동합니다                    |
| **소스** | URL / 캐릭터     | 직접 URL을 입력하거나, 등록된 캐릭터의 스프라이트에서 선택합니다 |
| **변형** | 감정별 스프라이트     | 기본, 행복, 슬픔 등 등록해 둔 감정 변형을 선택합니다       |

### 배경음악 (BGM)

| 항목     | 옵션                   | 설명                               |
| ------ | -------------------- | -------------------------------- |
| **모드** | 상속 / 설정 / 정지 / 페이드아웃 | BGM을 유지, 새로 재생, 정지, 서서히 줄이기 중 선택 |
| **음원** | YouTube URL          | 재생할 음악의 YouTube 링크를 입력합니다        |

> **주의:** BGM은 **YouTube 링크만** 사용할 수 있습니다. 오디오 파일 업로드는 지원하지 않습니다.

### 검수 도우미 활용하기

인스펙터 상단에는 **검수 도우미**가 있습니다. 현재 노드의 배경, BGM, 스프라이트가 어디서 상속되고 있는지를 실시간으로 보여줍니다.

"배경이 왜 안 나오지?", "BGM이 왜 안 멈추지?" 같은 문제가 생기면 검수 도우미를 확인해보세요.

{/*  IMAGE: validator-panel — 검수 도우미 패널. BGM/배경/스프라이트의 상속 상태가 표시된 모습  */}

검수 도우미가 감지하는 문제:

| 검사 항목                  | 심각도 |
| ---------------------- | --- |
| 연결되지 않은 선택지            | 오류  |
| 막다른 노드 (다음으로 진행할 수 없음) | 경고  |
| 엔딩 노드인데 나가는 연결이 있음     | 정보  |
| 빈 대사                   | 정보  |

> **이 단계의 결과:** 노드에 배경 이미지, BGM 등 연출이 설정된 상태입니다.

## Step 8. 엔딩 만들기

시나리오에는 **최소 1개의 엔딩**이 반드시 필요합니다.

### 8-1. 엔딩 노드를 지정합니다

스토리의 마지막 노드를 선택하고, 인스펙터에서 **"엔딩 노드"** 옵션을 활성화합니다.

{/*  IMAGE: ending-setting — 인스펙터에서 엔딩 노드를 설정하는 화면. 엔딩 유형, 제목, 설명 입력란이 보이는 상태  */}

### 8-2. 엔딩 정보를 입력합니다

| 항목            | 설명                                                                                                                                                                                                                |
| ------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **엔딩 유형**     | 트루엔딩 (최선의 결말) / 노멀엔딩 (일반적인 결말) / 배드엔딩 (좋지 않은 결말) / 히든엔딩 (숨겨진 결말)                                                                                                                                                  |
| **엔딩 제목**     | 엔딩의 이름 (예: "함께하는 내일")                                                                                                                                                                                             |
| **엔딩 설명**     | 엔딩에 대한 상세 설명입니다 (최대 1,000자). 이 텍스트는 엔딩 이후 **채팅 모드에서 AI 캐릭터의 시스템 프롬프트에 직접 주입**됩니다. 엔딩 직후의 상황, 캐릭터의 감정 상태, 관계 변화 등을 구체적으로 작성하면, AI가 해당 엔딩의 맥락에 맞는 자연스러운 대화를 생성합니다. 미입력 시 `"{엔딩 제목} 엔딩 이후 시점으로 대화합니다."`가 자동으로 사용됩니다. |
| **엔딩 루트**     | 이 엔딩과 연결된 캐릭터 (공략 대상)                                                                                                                                                                                             |
| **엔딩 이미지/영상** | 엔딩 화면에 표시되는 미디어. 이미지 5MB, 영상 20MB.                                                                                                                                                                                |

독자가 엔딩에 도달하면 **컬렉션**에 해금 기록이 남습니다.

> **이 단계의 결과:** 스토리의 마지막 노드가 엔딩으로 지정되어 있고, 시작부터 엔딩까지 도달할 수 있는 경로가 완성된 상태입니다.

## Step 9. 미리보기로 테스트하기

공개하기 전에 반드시 미리보기로 시나리오를 확인합니다.

### 9-1. 미리보기를 실행합니다

툴바에서 **미리보기(재생) 버튼**을 클릭하면, 실제 독자가 보는 화면과 동일한 형태로 시나리오가 재생됩니다.

{/*  IMAGE: preview-play — 미리보기 모드에서 시나리오를 플레이하는 화면  */}

### 9-2. 체크리스트를 확인합니다

미리보기를 하면서 다음 항목들을 확인하세요:

- 모든 선택지가 올바른 노드로 연결되는지
- 배경, 스프라이트, BGM이 의도대로 전환되는지
- 대사에 오탈자가 없는지
- 시작부터 엔딩까지 정상적으로 도달하는지
- 엔딩 외 막다른 노드(더 이상 진행할 수 없는 곳)가 없는지

> **이 단계의 결과:** 시나리오를 처음부터 끝까지 플레이해보고, 문제가 없는 것을 확인한 상태입니다.

## Step 10. 공개하기

시나리오가 완성되었으면 공개합니다. 별도의 심사 없이 **즉시 게시**됩니다.

### 10-1. 먼저 "비공개 플레이"로 테스트합니다 (권장)

처음이라면 **비공개 플레이** 옵션을 먼저 선택하는 것을 권장합니다. 비공개 플레이는 **본인만** 테스트 플레이할 수 있는 모드입니다. 충분히 테스트한 뒤, 준비가 되면 공개로 전환할 수 있습니다.

### 10-2. 준비가 되면 "공개하기"를 선택합니다

에디터 상단의 **제출하기** 버튼을 클릭하면, 다음 두 가지 옵션이 표시됩니다:

| 옵션          | 설명                                              |
| ----------- | ----------------------------------------------- |
| **공개하기**    | **즉시** 모든 독자가 플레이할 수 있게 됩니다. 피스 수익을 획득할 수 있습니다. |
| **비공개 플레이** | 본인만 테스트 플레이할 수 있습니다. 나중에 공개로 전환할 수 있습니다.        |

{/*  IMAGE: submit-dialog — 제출 다이얼로그. "공개하기"와 "비공개 플레이" 두 옵션이 보이는 상태  */}

### 공개 시 자동 검증 기준

공개하기를 선택하면, 시나리오가 아래 기준을 통과하는지 자동으로 검사됩니다. 기준을 충족하지 않으면 공개할 수 없습니다.

| 검사 항목  | 기준                | 분류 |
| ------ | ----------------- | --- |
| 노드 수   | 3개 이상, 200개 이하    | 필수 |
| 엔딩 수   | 1개 이상             | 필수 |
| 제목 길이  | 3자 이상, 100자 이하    | 필수 |
| 설명 길이  | 10자 이상            | 권장 |
| 설명 최대  | 500자 이하           | 필수 |
| 고립된 노드 | 시작점에서 도달할 수 없는 노드 | 경고 |

### 상태 흐름

```text
초안 (비공개)  →  공개하기  →  공개중 (즉시 게시)
                              ↕ 비공개 전환 가능
```

| 상태           | 설명                            | 가능한 작업                   |
| ------------ | ----------------------------- | ------------------------ |
| **초안 (비공개)** | 에디터에서 자유롭게 편집 가능. 본인만 플레이 가능. | 수정, 삭제, 공개하기             |
| **공개중**      | 모든 독자가 플레이 가능. 편집도 가능.        | 수정 후 저장 (바로 반영), 비공개로 전환 |

> **참고:** 공개 중인 시나리오가 커뮤니티 가이드라인을 위반하는 경우, 관리자에 의해 사후에 제한될 수 있습니다.

### 주의사항

- 하루 최대 **10건**까지 공개할 수 있습니다.
- 공개 중인 시나리오를 삭제하려면 먼저 **비공개로 전환**해야 합니다.
- 공개 중인 시나리오도 언제든 수정할 수 있으며, 수정 사항은 바로 반영됩니다.

> **이 단계의 결과:** 시나리오가 공개되어 모든 독자가 플레이할 수 있는 상태입니다. 수고하셨습니다!

## 이미지/에셋 상세 스펙

시나리오에 사용하는 모든 이미지/미디어의 상세 스펙을 정리합니다.

### 이미지

| 에셋             | 비율         | 포맷                   | 업로드 제한 | 서버 저장       | 비고              |
| -------------- | ---------- | -------------------- | ------ | ----------- | --------------- |
| **썸네일**        | 3:4 (세로형)  | JPEG, PNG, GIF, WebP | 5MB    | 자동 압축 → 2MB | 고화질 세로형 권장      |
| **캐릭터 대표 이미지** | 3:4 (세로형)  | JPEG, PNG, GIF, WebP | 5MB    | 자동 압축 → 2MB |                 |
| **스프라이트**      | 1:1 (정사각형) | **PNG 권장** (투명 배경)   | 5MB    | 자동 압축 → 2MB | 캐릭터당 최대 20장     |
| **배경 이미지**     | 가로형 권장     | JPEG, PNG, GIF, WebP | 5MB    | 자동 압축 → 2MB | 최대 1920px로 리사이즈 |
| **엔딩 이미지**     | 자유         | JPEG, PNG, GIF, WebP | 5MB    | 자동 압축 → 2MB |                 |

### 영상

| 에셋        | 포맷        | 업로드 제한 | 비고 |
| --------- | --------- | ------ | --- |
| **엔딩 영상** | MP4, WebM | 20MB   |    |

### 오디오

| 에셋      | 지원 방식           | 비고        |
| ------- | --------------- | --------- |
| **BGM** | **YouTube 링크만** | 파일 업로드 불가 |
| **효과음** | **YouTube 링크만** | 파일 업로드 불가 |

### 이미지 업로드 시 알아두면 좋은 점

- **자동 압축**: 5MB 이하의 이미지를 업로드하면, 서버에 저장될 때 자동으로 2MB 이하로 압축됩니다. 별도로 리사이즈하거나 압축할 필요가 없습니다.
- **최대 해상도**: 이미지의 가로 또는 세로 중 긴 쪽이 1920px을 넘으면 자동으로 리사이즈됩니다.
- **투명 배경 (스프라이트)**: 스프라이트 이미지는 캐릭터만 남기고 배경을 투명하게 처리한 **누끼 이미지**가 필요합니다. PNG 포맷을 사용해야 투명 배경이 유지됩니다.
- **5MB 초과 이미지**: 5MB를 초과하는 이미지는 업로드할 수 없습니다. 이미지 편집 도구로 미리 크기를 줄여주세요.

## 유용한 기능 모음

에디터에는 시나리오 제작을 편리하게 해주는 다양한 기능이 있습니다.

### 퀵 스타트 체크리스트

에디터 안에 내장된 **퀵 스타트 가이드** 패널을 활용하세요. 시나리오 제작에 필요한 항목들의 완료 여부를 자동으로 추적해줍니다.

진행률 바에서 "필수 N/9개 완료" 형태로 현재 상태를 확인할 수 있으며, 각 항목을 클릭하면 해당 기능으로 바로 이동할 수 있습니다.

{/*  IMAGE: quickstart-panel — 퀵스타트 체크리스트 패널. 카테고리별 진행률과 체크 항목이 보이는 상태  */}

### 찾기/바꾸기

시나리오의 **모든 노드**에 걸쳐 텍스트를 검색하고, 원하는 단어로 일괄 치환할 수 있습니다. 캐릭터 이름을 바꾸거나 오타를 수정할 때 유용합니다.

### 스프라이트 일괄 교체

여러 노드에 적용된 스프라이트를 **한 번에** 다른 스프라이트로 교체할 수 있습니다.

### 참조 스니펫 패널

에디터 내에서 **참조 스니펫 패널**을 열면, 다음과 같은 참조용 예시를 확인할 수 있습니다:

- **플로우 패턴**: 일직선 대화, 선택지 분기, 엔딩 구조, 분기-합류
- **캐릭터 예시**: 메인/서브 캐릭터 설정 예시
- **씬 예시**: 실내/실외 장면 설정 예시

스니펫을 참고하여 비슷한 구조를 직접 만들어보세요.

### 내보내기/가져오기

시나리오 데이터를 파일로 저장하거나 불러올 수 있습니다. 백업 또는 협업에 활용하세요. 특정 챕터의 데이터만 별도로 내보내는 것도 가능합니다.

### 보기 모드

에디터는 세 가지 보기 모드를 지원합니다. 상황에 따라 전환하며 사용하세요.

| 모드       | 설명                           | 용도         |
| -------- | ---------------------------- | ---------- |
| **그래프**  | 노드를 캔버스에 배치하고 연결하는 시각적 편집 모드 | 기본 작업 모드   |
| **리스트**  | 노드를 목록 형태로 나열하여 텍스트 중심으로 편집  | 대사 집중 편집 시 |
| **JSON** | 시나리오 데이터를 JSON 형태로 직접 편집     | 고급 사용자     |

### 되돌리기/다시하기

실수로 변경한 내용은 툴바의 **되돌리기/다시하기** 버튼으로 복원할 수 있습니다.

## 더 알아보기: 고급 기능

기본 시나리오를 완성한 후, 아래 기능들을 활용하면 더 풍성한 시나리오를 만들 수 있습니다.

### 챕터 구성하기

시나리오를 **챕터 단위**로 나눌 수 있습니다.

노드의 **챕터 제목** 필드에 값을 입력하면, 해당 노드가 새 챕터의 시작점이 됩니다. 수익화를 "부분 유료"로 설정한 경우, 챕터 단위로 유료 설정이 가능합니다 (첫 번째 챕터는 기본 무료).

### 시작 화면 커스터마이징

독자가 시나리오를 시작할 때 보게 되는 **타이틀 화면**을 꾸밀 수 있습니다.

| 항목     | 설명        |
| ------ | --------- |
| 메인 타이틀 | 시나리오 제목   |
| 서브 타이틀 | 부제        |
| 설명     | 간단한 소개 문구 |

### 수익화 설정

시나리오 설정의 **수익화** 탭에서 과금 방식을 설정할 수 있습니다.

| 유형        | 설명                                      |
| --------- | --------------------------------------- |
| **완전 무료** | 모든 챕터를 무료로 제공                           |
| **부분 유료** | 일부 챕터를 유료로 설정 (기본 챕터 가격, 무료 챕터 수 설정 가능) |

수익화에 대한 자세한 내용은 [수익 창출 가이드](/monetization-guide)를 참고하세요.

## 자주 묻는 질문

### Q. 시나리오에 몇 개의 노드까지 만들 수 있나요?

최소 3개, 최대 200개의 노드를 포함할 수 있습니다.

### Q. 시나리오에 캐릭터를 몇 명까지 등록할 수 있나요?

공략 캐릭터 수는 최대 20명까지 설정 가능합니다.

### Q. 저장은 어떻게 하나요?

시나리오 에디터는 **자동 저장**됩니다. 변경사항이 있을 때마다 자동으로 서버에 반영되며, 저장 완료 시 "저장됨" 표시가 나타납니다.

### Q. 모바일에서도 편집할 수 있나요?

시나리오 에디터는 데스크탑 전용입니다. 모바일에서는 시나리오 정보를 확인할 수는 있지만 편집은 불가능합니다.

### Q. 공개 중인 시나리오를 수정할 수 있나요?

가능합니다. 수정 후 저장하면 바로 반영됩니다.

### Q. 비공개로 만든 시나리오를 나중에 공개할 수 있나요?

가능합니다. 비공개 상태의 시나리오는 언제든 "공개하기"로 전환하여 즉시 게시할 수 있습니다.

### Q. 공개 중인 시나리오를 삭제할 수 있나요?

공개 중인 시나리오는 바로 삭제할 수 없습니다. 먼저 비공개로 전환한 뒤 삭제해야 합니다.

### Q. 비공개 시나리오도 직접 플레이할 수 있나요?

가능합니다. 비공개 시나리오는 본인만 플레이할 수 있으며, 테스트 목적으로 활용하시면 됩니다.

### Q. 찾기/바꾸기 기능은 어떤 범위에서 작동하나요?

시나리오의 모든 노드에 걸쳐 텍스트를 검색하고 일괄 치환할 수 있습니다.

### Q. 이미지가 업로드되지 않아요.

다음을 확인해보세요:

- **파일 크기**: 이미지는 5MB 이하, 영상은 20MB 이하여야 합니다.
- **파일 포맷**: JPEG, PNG, GIF, WebP만 지원합니다.

### Q. BGM은 어떻게 넣나요?

BGM은 **YouTube 링크**로만 추가할 수 있습니다. 오디오 파일 업로드는 지원하지 않습니다. 인스펙터의 BGM 설정에서 YouTube URL을 입력하세요.

### Q. 이미지 자동 압축이 되나요?

네. 5MB 이하의 이미지를 업로드하면 자동으로 2MB 이하로 압축됩니다. 가로/세로 중 긴 쪽이 1920px을 넘으면 자동으로 리사이즈됩니다. 별도로 리사이즈하거나 압축할 필요가 없습니다.
