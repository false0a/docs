## 1. 시나리오란?

루벳 시나리오는 **비주얼 노벨** 형식의 인터랙티브 스토리입니다. 사용자는 스토리를 읽으며 선택지를 고르고, 선택에 따라 달라지는 스토리 전개와 다양한 엔딩을 경험합니다.

시나리오는 **노드 기반 플로우차트**로 구성됩니다. 각 노드는 대사, 선택지, 연출 등 하나의 장면 단위이며, 노드를 연결하여 전체 스토리의 흐름을 만듭니다.

## 2. 시작하기

### 새 시나리오 만들기

1. **제작 스튜디오**로 이동합니다.
2. "내 시나리오" 섹션에서 **새 시나리오** 버튼을 클릭합니다.
3. 자동으로 제목이 생성되며 (예: "새 시나리오 1"), 시나리오 에디터로 이동합니다.

> 시나리오 에디터는 **데스크탑 전용**입니다. 모바일에서는 편집이 불가능합니다.

### 템플릿 선택

새 시나리오를 만들면 빈 템플릿으로 시작합니다. 에디터 내에서 **템플릿 참조 패널**을 열면 다음과 같은 참조 스니펫을 삽입할 수 있습니다.

- **플로우 패턴**: 일직선 대화, 선택지 분기, 엔딩 구조, 분기-합류
- **캐릭터 스니펫**: 메인/서브 캐릭터 예시
- **씬 스니펫**: 실내/실외 장면 예시

## 3. 에디터 화면 구성

### 보기 모드

에디터는 세 가지 보기 모드를 지원합니다.

| 모드   | 설명                               | 용도       |
| ---- | -------------------------------- | -------- |
| 그래프  | 노드를 캔버스에 배치하고 연결하는 시각적 편집 모드     | 주요 작업 모드 |
| 리스트  | 노드를 목록 형태로 나열하여 텍스트 중심으로 편집하는 모드 | 대사 집중 편집 |
| JSON | 시나리오 데이터를 JSON 형태로 직접 편집하는 모드    | 고급 사용자용  |

### 상단 툴바

에디터 상단에 다음 도구들이 배치되어 있습니다.

| 도구          | 설명                         |
| ----------- | -------------------------- |
| 노드 추가 (+)   | 새 노드를 캔버스에 추가              |
| 캐릭터 관리      | 시나리오에 등장하는 캐릭터 설정          |
| 미리보기        | 시나리오를 플레이어 시점에서 재생         |
| 찾기/바꾸기      | 모든 노드에서 텍스트 검색 및 치환        |
| 스프라이트 일괄 교체 | 여러 노드의 스프라이트를 한 번에 변경      |
| 대사 일괄 가져오기  | 서식화된 텍스트를 붙여넣어 대사를 한 번에 입력 |
| 에셋 탐색기      | 이미지, 배경, 음악 등의 에셋 탐색 및 업로드 |
| 설정          | 시나리오 기본 정보, 수익화, 시작 화면 설정  |
| 내보내기/가져오기   | 시나리오 데이터를 파일로 저장하거나 불러오기   |
| 그룹 추가       | 노드를 시각적으로 묶는 그룹 생성         |
| 되돌리기/다시하기   | 편집 기록 되돌리기 및 복원            |
| 퀵 스타트 가이드   | 제작 체크리스트 패널                |
| 템플릿 참조      | 참조용 스니펫 패널                 |

### 우측 패널 (인스펙터)

캔버스에서 노드를 선택하면 우측에 **인스펙터** 패널이 열립니다. 선택한 노드의 대사, 설정, 연출 등을 편집할 수 있습니다.

### 검수 도우미

인스펙터에는 실시간으로 시나리오 문제를 감지하는 **검수 도우미**가 포함되어 있습니다.

| 검사 항목             | 심각도 |
| ----------------- | --- |
| 연결되지 않은 선택지       | 오류  |
| 막다른 노드 (진행 불가)    | 경고  |
| 엔딩 노드에 나가는 연결이 있음 | 정보  |
| 빈 대사              | 정보  |

## 4. 노드와 플로우 이해하기

### 노드란?

노드는 시나리오의 **최소 장면 단위**입니다. 하나의 노드에는 여러 줄의 대사, 배경 설정, 스프라이트 표시, 효과음 등이 포함될 수 있습니다.

### 노드 추가

- 툴바의 **\+** 버튼을 클릭하거나
- 캔버스의 빈 공간을 더블클릭합니다.

### 노드 연결

- 노드의 출력 핸들(하단)에서 드래그하여 다른 노드의 입력 핸들(상단)에 연결합니다.
- 선택지가 있는 노드는 각 선택지마다 별도의 연결을 설정합니다.

### 시작 노드

시나리오의 **첫 번째 노드**를 시작 노드로 지정해야 합니다. 인스펙터에서 "시작 노드로 지정" 옵션을 설정할 수 있습니다.

### 플로우 구조 예시

**일직선 구조:**

```
시작 → 대사1 → 대사2 → 대사3 → 엔딩
```

**분기 구조:**

```
시작 → 선택지 → [선택A] → 루트A → 엔딩A
                → [선택B] → 루트B → 엔딩B
```

**합류 구조:**

```
시작 → 선택지 → [선택A] → 합류 지점 → 공통 진행 → 엔딩
                → [선택B] ↗
```

## 5. 대사 작성하기

노드를 선택하면 인스펙터에서 대사를 편집할 수 있습니다.

### 대사 시퀀스

하나의 노드 안에 **여러 줄의 대사**를 순서대로 배치할 수 있습니다. 각 줄은 독립적인 대사 단위입니다.

### 대사 줄 구성 요소

| 요소     | 설명                    |
| ------ | --------------------- |
| 캐릭터 선택 | 이 대사를 말하는 캐릭터를 지정합니다  |
| 메시지 유형 | 대사 / 나레이션 / 연출 중 선택   |
| 텍스트    | 실제 대사 또는 나레이션 내용      |
| 속마음    | 캐릭터의 내면 묘사 (이탤릭체로 표시) |
| 스프라이트  | 이 대사에서 표시할 캐릭터 일러스트   |
| 효과음    | 이 대사와 함께 재생할 효과음      |

### 메시지 유형

| 유형               | 설명                   | 표시 방식     |
| ---------------- | -------------------- | --------- |
| 대사 (dialogue)    | 캐릭터가 직접 말하는 대화       | 말풍선 형태    |
| 나레이션 (narration) | 상황 설명, 심리 묘사         | 화면 전체 텍스트 |
| 연출 (stage)       | 장면 전환, 시간 경과 등 연출 지시 | 연출 표시     |

### 대사 줄 조작

- **순서 변경**: 드래그 핸들로 위/아래 이동
- **위/아래 이동**: 화살표 버튼
- **노드 분할**: 특정 줄에서 노드를 둘로 분리
- **삭제**: 휴지통 버튼
- **일괄 선택**: 체크박스로 여러 줄을 선택하여 이동/삭제

### 대사 일괄 가져오기

툴바의 "대사 일괄 가져오기" 기능을 사용하면, 서식화된 텍스트를 붙여넣어 한 번에 여러 대사를 입력할 수 있습니다.

## 6. 선택지 만들기

선택지는 사용자가 스토리의 방향을 결정하는 핵심 요소입니다.

### 선택지 추가

인스펙터의 **선택지** 섹션에서 선택지를 추가합니다.

### 선택지 구성 요소

| 요소     | 설명                                |
| ------ | --------------------------------- |
| 텍스트    | 사용자에게 표시되는 선택지 문구                 |
| 다음 노드  | 이 선택지를 고르면 이동할 노드                 |
| 호감도 변화 | 선택 시 캐릭터별 호감도 증감 값                |
| 조건     | 이 선택지가 표시되기 위한 조건 (호감도 범위, 플래그 등) |
| 플래그 설정 | 선택 시 설정/해제할 플래그                   |

### 조건부 선택지

특정 조건을 만족할 때만 표시되는 선택지를 만들 수 있습니다.

| 조건 유형  | 설명                        |
| ------ | ------------------------- |
| 최소 호감도 | 특정 캐릭터의 호감도가 일정 수치 이상일 때  |
| 최대 호감도 | 특정 캐릭터의 호감도가 일정 수치 이하일 때  |
| 플래그    | 이전 선택에서 설정한 플래그가 활성화되었을 때 |

### 호감도 변화

각 선택지마다 캐릭터별 호감도 변화를 설정할 수 있습니다. 양수(+)는 호감도 증가, 음수(-)는 호감도 감소를 의미합니다.

## 7. 캐릭터 등록 및 관리

시나리오에 등장하는 캐릭터를 등록합니다.

### 캐릭터 등록 방법

툴바의 **캐릭터 관리** 버튼을 클릭하여 캐릭터를 추가/수정합니다.

### 캐릭터별 설정 항목

| 항목     | 설명             |
| ------ | -------------- |
| 이름     | 캐릭터 이름         |
| 역할     | 메인 / 서브 / 엑스트라 |
| 대표 이미지 | 캐릭터 기본 일러스트    |
| 스프라이트  | 감정/상황별 일러스트 매핑 |

시나리오에 등록된 캐릭터는 **제작 스튜디오의 캐릭터**와 연결됩니다. 기존에 만든 캐릭터를 시나리오에 배치하거나, 시나리오 안에서 새로 만들 수 있습니다.

## 8. 배경과 연출 설정

각 노드에 배경, 효과음, 화면 효과 등을 설정하여 시각적/청각적 연출을 더할 수 있습니다.

### 배경 설정

| 항목     | 옵션                        | 설명                                   |
| ------ | ------------------------- | ------------------------------------ |
| 모드     | 상속 / 설정 / 숨기기             | 이전 노드의 배경을 그대로 사용하거나, 새로 지정하거나, 숨깁니다 |
| 배경 이미지 | 에셋 탐색기에서 선택               | 배경으로 사용할 이미지를 지정합니다                  |
| 시간 필터  | 없음 / 낮 / 석양 / 밤 / 새벽 / 흐림 | 같은 배경에 시간대별 색감 필터를 적용합니다             |

### 스프라이트 설정

| 항목 | 옵션             | 설명                                  |
| --- | -------------- | ----------------------------------- |
| 모드 | 상속 / 설정 / 숨기기  | 이전 노드의 스프라이트를 유지하거나, 새로 지정하거나, 숨깁니다 |
| 소스 | URL / 캐릭터      | 직접 URL을 입력하거나, 캐릭터의 스프라이트에서 선택합니다   |
| 변형 | (캐릭터 스프라이트 목록) | 기본, 행복, 슬픔 등 감정별 변형을 선택합니다          |

### 배경음악 (BGM)

| 항목 | 옵션                   | 설명                               |
| --- | -------------------- | -------------------------------- |
| 모드 | 상속 / 설정 / 정지 / 페이드아웃 | BGM을 유지, 새로 재생, 정지, 서서히 줄이기 중 선택 |
| 음원 | 에셋 또는 URL            | 재생할 음악 파일을 지정합니다                 |

## 9. 엔딩 설정하기

시나리오의 결말을 정의합니다.

### 엔딩 노드 만들기

인스펙터에서 노드를 **엔딩 노드**로 지정합니다.

### 엔딩 유형

| 유형   | 설명                       |
| ---- | ------------------------ |
| 트루엔딩 | 최선의 결말                   |
| 노멀엔딩 | 일반적인 결말                  |
| 배드엔딩 | 좋지 않은 결말                 |
| 히든엔딩 | 특수 조건으로만 도달할 수 있는 숨겨진 결말 |

### 엔딩 설정 항목

| 항목    | 설명                                       |
| ----- | ---------------------------------------- |
| 엔딩 제목 | 엔딩의 이름                                   |
| 엔딩 설명 | 엔딩에 대한 간단한 설명                            |
| 엔딩 루트 | 이 엔딩과 연결된 캐릭터 (공략 대상)                    |
| 엔딩 영상 | 엔딩 화면에 표시되는 이미지 또는 영상 (이미지 5MB, 영상 20MB) |

사용자가 엔딩에 도달하면 **컬렉션**에 해금 기록이 남습니다.

### 제작 기준

시나리오를 공개 제출하려면 **최소 1개의 엔딩**이 필요합니다.

---

## 10. 챕터 구성하기

시나리오를 챕터 단위로 나누어 구성할 수 있습니다.

### 챕터 시작점 지정

노드의 **챕터 제목** 필드에 값을 입력하면, 해당 노드가 새 챕터의 시작점이 됩니다.

### 챕터와 수익화

수익화를 "부분 유료"로 설정한 경우, 챕터 단위로 유료 설정이 가능합니다.

- 기본적으로 첫 번째 챕터는 무료입니다 (무료 챕터 수 설정 가능).
- 이후 챕터는 기본 가격이 적용되거나, 개별 노드에서 **챕터 가격**을 직접 지정할 수 있습니다.

## 11. 씬(장면) 관리

씬은 **장소와 상황**을 정의하는 단위입니다.

### 씬 구성 요소

| 항목     | 설명                            |
| ------ | ----------------------------- |
| 이름     | 씬 이름 (예: "교실", "카페", "저택 정원") |
| 설명     | 씬에 대한 간단한 설명                  |
| 장소     | 씬의 위치                         |
| 시간     | 아침 / 오후 / 저녁 / 밤 등            |
| 배경 이미지 | 기본 배경 이미지                     |
| BGM    | 기본 배경음악                       |
| 등장 캐릭터 | 이 씬에 등장하는 캐릭터 목록              |

씬을 노드에 연결하면, 해당 씬의 배경과 BGM이 자동으로 적용됩니다.

노드가 많아지면 **그룹** 기능을 활용하여 시각적으로 정리할 수 있습니다.

툴바의 **그룹 추가** 버튼을 클릭하거나, 여러 노드를 선택한 후 그룹으로 묶습니다.

| 항목     | 설명                                                      |

### 미리보기

- 엔딩까지 정상적으로 도달하는지

## 16. 제출 및 심사

시나리오를 다른 사용자에게 공개하려면 관리자 심사를 거쳐야 합니다.

### 심사 흐름

```
초안 (비공개)  →  공개 제출  →  심사 대기  →  승인됨 / 거절됨
```

### 제출 방법

1. 에디터 상단의 **제출하기** 버튼을 클릭합니다.
2. 두 가지 옵션 중 선택합니다:
   - **공개 제출**: 심사 후 모든 사용자가 플레이 가능, 피스 수익 획득 가능
   - **비공개 플레이**: 본인만 테스트 플레이 가능, 나중에 공개 전환 가능

### 상태별 안내

| 상태       | 설명               | 가능한 작업     |
| -------- | ---------------- | ---------- |
| 초안 (비공개) | 에디터에서 자유롭게 편집 가능 | 수정, 삭제, 제출 |
| 심사 대기 중  | 관리자가 검토 중, 수정 불가 | 제출 취소      |
| 승인됨      | 공개된 상태, 편집 가능    | 수정 후 저장    |
| 거절됨      | 거절 사유가 표시됨       | 수정 후 재제출   |

### 심사 기준 (제출 시 자동 검사)

| 검사 항목  | 기준                | 분류 |
| ------ | ----------------- | --- |
| 노드 수   | 3개 이상, 200개 이하    | 필수 |
| 엔딩 수   | 1개 이상             | 필수 |
| 제목 길이  | 3자 이상, 100자 이하    | 필수 |
| 설명 길이  | 10자 이상            | 권장 |
| 설명 최대  | 500자 이하           | 필수 |
| 고립된 노드 | 시작점에서 도달할 수 없는 노드 | 경고 |
| 금지어    | 금지 단어 포함 여부       | 필수 |

### 주의사항

- 심사 중에는 시나리오를 수정할 수 없습니다.
- 하루 최대 **10건**까지 제출할 수 있습니다.
- 승인된 시나리오는 삭제할 수 없습니다.
- 거절된 경우 사유를 확인하고 수정 후 다시 제출하세요.

## 17. 퀵 스타트 체크리스트

에디터 내 **퀵 스타트 가이드**에서 제작 진행 상황을 한눈에 확인할 수 있습니다. 각 항목은 자동으로 완료 여부가 감지됩니다.

### 기본 정보 (필수)

- 시나리오 제목 설정
- 상세 설명 작성 (10자 이상)
- 썸네일 이미지 등록
- 태그 설정 (선택)

### 캐릭터 설정 (필수)

- 캐릭터 1명 이상 등록
- 역할(메인/서브/엑스트라) 지정
- AI 페르소나 프롬프트 작성
- 스프라이트 이미지 설정 (선택)
- 호감도 단계 설정 (선택)

### 스토리 작성 (필수)

- 시작 노드 지정
- 대사 작성
- 선택지 추가
- 씬(배경) 설정 (선택)

### 마무리 (권장)

- 시작 화면 설정 (선택)

- 플레이타임 설정 (선택)

## 18. 자주 묻는 질문

### Q. 시나리오에 몇 개의 노드까지 만들 수 있나요?

최소 3개, 최대 200개의 노드를 포함할 수 있습니다.

### Q. 시나리오에 캐릭터를 몇 명까지 등록할 수 있나요?

공략 캐릭터 수는 최대 20명까지 설정 가능합니다.

### Q. 저장은 어떻게 하나요?

시나리오 에디터는 **자동 저장**됩니다. 변경사항이 있을 때마다 자동으로 서버에 반영되며, 저장 완료 시 "저장됨" 표시가 나타납니다.

### Q. 모바일에서도 편집할 수 있나요?

시나리오 에디터는 데스크탑 전용입니다. 모바일에서는 시나리오 정보를 확인할 수는 있지만 편집은 불가능합니다.

### Q. 승인된 시나리오를 수정할 수 있나요?

가능합니다. 수정 후 저장하면 바로 반영됩니다.

### Q. 비공개로 만든 시나리오도 직접 플레이할 수 있나요?

가능합니다. 비공개 시나리오는 본인만 플레이할 수 있으며, 테스트 목적으로 활용하시면 됩니다.

### Q. 찾기/바꾸기 기능은 어떤 범위에서 작동하나요?

시나리오의 모든 노드에 걸쳐 텍스트를 검색하고 일괄 치환할 수 있습니다.

### Q. 그룹은 스토리에 영향을 주나요?

아닙니다. 그룹은 에디터 화면에서 노드를 시각적으로 정리하는 도구일 뿐, 실제 플레이에는 아무 영향을 주지 않습니다.
