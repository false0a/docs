# 시나리오 작성 가이드

루벳의 시나리오 에디터를 사용하면, 코딩 없이도 이용자가 직접 선택하고 참여하는 비주얼 노벨 스토리를 만들 수 있습니다. 이 문서에서는 에디터 사용법과 시나리오 제작에 필요한 모든 설정을 안내합니다.

---

## 에디터 개요

시나리오 에디터는 **노드(블록)** 를 연결하여 이야기의 흐름을 만드는 방식입니다. 각 노드는 대사, 선택지, 엔딩, 효과 등 하나의 장면 단위를 담당합니다.

### 세 가지 보기 모드

| 보기 모드 | 설명 |
|-----------|------|
| 그래프 | 노드와 연결선을 시각적으로 보여주는 기본 모드. 드래그로 노드를 옮기고 선으로 연결할 수 있습니다 |
| 리스트 | 노드를 목록 형태로 나열해서 빠르게 훑어볼 수 있는 모드 |
| 데이터 보기 | 시나리오 원본 데이터를 직접 확인하고 편집할 수 있는 고급 모드 |

---

## 노드 종류

### 대사 노드

캐릭터의 대사, 지문, 나레이션 등을 담는 가장 기본적인 노드입니다. 하나의 노드 안에 **여러 줄의 대사**를 순서대로 넣을 수 있습니다.

각 대사 줄마다 다음을 설정할 수 있습니다:

| 항목 | 설명 |
|------|------|
| 캐릭터 | 이 대사를 말하는 캐릭터 (지문/나레이션이면 비워둡니다) |
| 대사 유형 | 대사, 나레이션, 시스템 메시지, 무대 지시 중 선택 |
| 대사 내용 | 캐릭터가 말하는 텍스트 |
| 속마음 | 캐릭터의 내면 독백 (이용자에게 따로 표시됩니다) |
| 표정/포즈 | 이 대사에서 표시할 스프라이트 변형 |
| 효과음 | 이 대사와 함께 재생할 소리 |
| 감정 | 대사의 감정 태그 |

<Note>
하나의 노드에 여러 대사를 넣으면 이용자가 한 장면씩 넘기며 읽게 됩니다. 장면 전환이나 분위기가 바뀌는 시점에서 새 노드를 만들면 관리가 편해집니다.
</Note>

#### 대사 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| 대사 | 캐릭터가 직접 말하는 대사 | "오늘 날씨가 좋네." |
| 나레이션 | 상황을 설명하는 서술 | 바람이 불어 커튼이 흔들렸다. |
| 시스템 메시지 | 시스템이 전달하는 안내문 | [저장 포인트] |
| 무대 지시 | 연출 지시 사항 | (주인공, 천천히 돌아본다) |

### 선택지 노드

이용자에게 선택지를 보여주고, 선택에 따라 다른 경로로 분기합니다.

각 선택지마다 설정 가능한 항목:

| 항목 | 설명 |
|------|------|
| 선택지 텍스트 | 이용자에게 표시되는 문구 |
| 다음 노드 | 선택 시 이동할 노드 |
| 호감도 변화 | 선택 시 특정 캐릭터와의 호감도 증감 |

### 채팅 모드 노드

비주얼 노벨 진행 중 **AI 자유 대화 구간**으로 전환됩니다. 이용자가 캐릭터와 직접 대화를 나누다가, 정해진 조건을 충족하면 다시 스토리로 돌아옵니다.

### 챕터 노드

챕터 제목과 가격을 지정하여 시나리오를 챕터 단위로 구분합니다. 유료 챕터 설정도 여기서 합니다.

### 엔딩 노드

시나리오의 종료 지점입니다. 하나의 시나리오에 여러 엔딩을 만들 수 있습니다.

### 효과 노드

화면 효과, 배경 전환, 사운드 재생 등 연출을 담당합니다.

### 자동 진행 노드

일정 시간이 지나면 자동으로 다음 노드로 넘어가는 타이머 노드입니다.

### 트리거 노드

이용자가 특정 문구를 입력하면 숨겨진 분기로 이동합니다.

## 속성 패널

그래프에서 노드를 클릭하면 오른쪽에 **속성 패널**이 열립니다. 여기서 선택한 노드의 모든 설정을 편집합니다.

### 패널 구성

| 섹션 | 내용 |
|------|------|
| 기본 설정 | 노드 제목, 챕터 제목, 챕터 가격 |
| 대사 목록 | 대사 편집. 드래그로 순서 변경, 일괄 가져오기, 다중 선택 지원 |
| 연출 및 소재 | 배경, 화면 전환, 화면 효과, 배경 음악 설정 |
| 선택지 및 분기 | 선택지 텍스트, 이동할 노드, 호감도 변화 |
| AI 채팅 상세 설정 | 채팅 모드 노드에서만 표시. AI 상황 지침, 최대 대화 횟수, 종료 조건 |
| 엔딩 설정 | 엔딩 유형, 제목, 설명, 관련 캐릭터, 엔딩 영상 |

---

## 채팅 모드 상세 설정

채팅 모드 노드를 사용하면, 비주얼 노벨 진행 도중 이용자가 **AI 캐릭터와 자유롭게 대화**하는 구간을 만들 수 있습니다.

### 설정 항목

| 항목 | 설명 |
|------|------|
| 첫 메시지 | 채팅 구간이 시작될 때 캐릭터가 먼저 보내는 메시지 |
| 시스템 지침 | AI에게 전달되는 상황 설명 (이용자에게는 보이지 않음) |
| 창의성 | AI 응답의 다양성 수준 (낮을수록 정해진 대로, 높을수록 자유롭게) |
| 최대 대화 횟수 | 채팅 구간에서 주고받을 수 있는 최대 메시지 수 |

### 종료 조건

채팅 구간이 끝나는 조건을 설정합니다.

| 종료 조건 | 설명 |
|-----------|------|
| 대화 횟수 제한 | 정해진 횟수만큼 대화하면 자동 종료 |
| 호감도 도달 | 캐릭터와의 호감도가 지정한 수치에 도달하면 종료 |
| 키워드 | 대화 중 특정 키워드가 등장하면 종료 |

### 성공 / 실패 분기

종료 조건 충족 여부에 따라 **성공 경로**와 **실패 경로**를 다른 노드로 연결할 수 있습니다.

- 예: 호감도 도달 조건에서, 호감도를 채우면 "고백 성공" 노드로, 대화 횟수가 먼저 소진되면 "아쉬운 이별" 노드로 분기

## 엔딩 설정

시나리오에는 **최소 1개**의 엔딩이 필요하며, 최대 **20개**까지 만들 수 있습니다.

### 엔딩 유형

| 유형 | 설명 |
|------|------|
| 트루 엔딩 | 시나리오의 가장 이상적인 결말 |
| 노멀 엔딩 | 보통의 결말 |
| 배드 엔딩 | 좋지 않은 결말 |
| 히든 엔딩 | 특수 조건을 만족해야 볼 수 있는 숨겨진 결말 |

### 엔딩에서 설정할 수 있는 항목

| 항목 | 설명 |
|------|------|
| 엔딩 제목 | 결말 화면에 표시되는 제목 |
| 엔딩 설명 | 결말에 대한 간략한 설명 |
| 관련 캐릭터 | 이 엔딩과 관련된 캐릭터 |
| 엔딩 이미지 | 결말 화면에 표시되는 이미지 |
| 엔딩 영상 | 결말 화면에 재생되는 영상 |
| 크레딧 | 제작진 정보 |
| 배경 음악 | 엔딩 화면에서 재생되는 음악 |

---

## 챕터와 수익화 설정

시나리오를 챕터 단위로 나누고, 특정 챕터를 유료로 설정하여 수익을 창출할 수 있습니다.

### 수익화 유형

| 유형 | 설명 |
|------|------|
| 무료 | 모든 챕터를 무료로 제공 |
| 유료 | 일부 또는 전체 챕터를 유료로 설정 |

### 유료 설정 방법

1. 시나리오 설정에서 수익화 유형을 **유료**로 선택합니다.
2. **무료 제공 챕터 수**를 정합니다. (예: 처음 2챕터는 무료)
3. **기본 챕터 가격**을 설정합니다.
4. 필요하면 노드별로 개별 챕터 가격을 지정할 수도 있습니다.

<Info>
첫 챕터를 무료로 제공하면 이용자가 이야기를 맛본 뒤 유료 챕터를 구매할 확률이 높아집니다.
</Info>

수익화에 대한 자세한 내용은 [수익 창출 가이드](/mintlify/monetization-guide)를 참고하세요.

## 씬 시스템

씬(Scene)은 시나리오 안에서 **장소와 분위기가 같은 장면 묶음**입니다. 씬을 활용하면 배경, 음악, 등장 캐릭터를 일괄로 관리할 수 있습니다.

### 씬에서 설정할 수 있는 항목

| 항목 | 설명 |
|------|------|
| 씬 이름 | 장면의 이름 (예: "카페", "학교 옥상") |
| 장소 | 장면이 벌어지는 장소 |
| 시간대 | 장면의 시간 (아침, 낮, 저녁, 밤 등) |
| 배경 이미지 | 이 씬의 기본 배경 |
| 배경 음악 | 이 씬에서 재생되는 기본 BGM |
| 등장 캐릭터 | 이 씬에 출연하는 캐릭터 목록 |
| 캐릭터별 스프라이트 매핑 | 캐릭터마다 이 씬에서 사용할 기본 이미지 지정 |
| 기본 장면 전환 | 이 씬에 들어올 때의 기본 전환 효과 |
| 보조 배경 | 같은 씬 안에서 전환할 수 있는 추가 배경들 |
| 진입 시 자동 연출 | 씬에 들어올 때 자동으로 실행할 연출 명령 |
| 퇴장 시 자동 연출 | 씬에서 나갈 때 자동으로 실행할 연출 명령 |

## 연출 및 효과

### 장면 전환 효과

노드가 바뀔 때 화면이 어떻게 전환되는지를 설정합니다.

| 효과 | 설명 |
|------|------|
| 없음 | 즉시 전환 |
| 페이드 | 서서히 어두워졌다 밝아짐 |
| 슬라이드 (왼쪽/오른쪽/위) | 화면이 밀려나며 전환 |
| 디졸브 (녹아들기) | 이전 장면이 녹아들 듯 사라짐 |
| 아이리스 (원형 전환) | 원형으로 좁혀지거나 넓어지며 전환 |

### 화면 효과

특정 순간에 화면에 특수 효과를 줄 수 있습니다.

| 효과 | 설명 |
|------|------|
| 흔들림 | 화면이 흔들리는 효과 (충격, 지진 등) |
| 백색 플래시 | 화면이 하얗게 번쩍이는 효과 |
| 적색 플래시 | 화면이 붉게 번쩍이는 효과 (위험, 충격 등) |
| 화면 가장자리 어둡게 | 화면 가장자리가 어두워지는 효과 (집중, 긴장 등) |
| 블러 | 화면이 흐릿해지는 효과 (회상, 기억 등) |
| 심장 박동 | 화면이 심장 박동처럼 확대/축소되는 효과 |

### 배경 시간 필터

같은 배경 이미지를 시간대에 따라 다른 분위기로 표현할 수 있습니다. 이미지를 여러 장 준비하지 않아도 됩니다.

| 필터 | 분위기 |
|------|--------|
| 없음 | 원본 그대로 |
| 낮 | 밝고 선명한 색감 |
| 석양 | 따뜻한 주황빛 톤 |
| 밤 | 어둡고 푸른 톤 |
| 새벽 | 차분한 분홍빛 톤 |
| 흐림 | 흐리고 탁한 톤 |

### 배경 모드

| 모드 | 설명 |
|------|------|
| 이전 장면 유지 | 앞 노드의 배경을 그대로 사용 |
| 직접 설정 | 새로운 배경 이미지를 지정 |
| 배경 숨기기 | 배경 없이 표시 |

### BGM (배경 음악) 설정

| 모드 | 설명 |
|------|------|
| 이전 곡 유지 | 앞 노드의 BGM을 계속 재생 |
| 직접 설정 | 새로운 음악 파일 지정 + 볼륨 조절 |
| 음악 정지 | BGM 재생 중단 |
| 서서히 사라짐 | 현재 음악이 점점 작아지며 중단 |

## 스프라이트 설정 (노드별)

각 노드에서 캐릭터 이미지가 어떻게 표시되는지를 세밀하게 조정할 수 있습니다.

| 항목 | 설명 |
|------|------|
| 스프라이트 이미지 | 이 노드에서 표시할 캐릭터 이미지 |
| 표시 방식 | 이전 유지 / 직접 설정 / 숨기기 |
| 이미지 출처 | 캐릭터 기본 이미지 / 직접 업로드 |
| 표정/포즈 | 캐릭터의 스프라이트 변형 선택 |
| 자동 매칭 | 대화 내용에 따라 표정을 자동으로 바꿀지 여부 |
| 레이어 색상 | 스프라이트 위에 입힐 색상 오버레이 |
| 레이어 투명도 | 색상 오버레이의 투명도 |

## 자동 연출 명령

노드가 표시될 때 **자동으로 실행되는 연출 명령 목록**을 설정할 수 있습니다. 복잡한 연출을 세밀하게 제어할 때 유용합니다.

### 사용할 수 있는 연출 명령

| 연출 명령 | 설명 |
|----------|------|
| 대기 | 지정한 시간만큼 멈춤 |
| 화면 효과 | 흔들림, 플래시, 블러 등 화면 효과 재생 |
| 효과음 재생 | 짧은 효과음 재생 |
| 배경 음악 | BGM 변경 또는 정지 |
| 진동 | 기기 진동 (모바일) |
| 프로필 변경 | 캐릭터 프로필 정보 변경 |
| 화면 알림 | 화면에 잠깐 뜨는 알림 표시 |
| 씬 전환 | 다른 씬으로 이동 |
| 보조 배경 전환 | 같은 씬 안에서 보조 배경으로 변경 |
| 스프라이트 변경 | 캐릭터 이미지를 다른 표정/포즈로 교체 |
| 배경 변경 | 배경 이미지 교체 |
| 호감도 변경 | 특정 캐릭터와의 호감도 증감 |

## 템플릿 활용하기

새 시나리오를 처음부터 만들기 어려울 때, 미리 준비된 템플릿으로 시작할 수 있습니다.

### 빈 템플릿

- 시작 노드 1개만 있는 가장 기본적인 템플릿입니다.
- 모든 설정을 직접 구성하고 싶을 때 선택하세요.
- 기본 설정: 장르 없음, 짧은 플레이 시간, 입문 난이도, 전체 이용가

### 로맨스 템플릿

- **5개의 노드**와 선택지 분기가 미리 구성되어 있습니다.
- 카페에서 시작하는 로맨스 시나리오 기본 틀을 제공합니다.
- 기본 캐릭터 1명("상대방")과 씬 1개("카페")가 포함됩니다.
- 기본 설정: 로맨스 장르, 중간 플레이 시간, 입문 난이도, 12세 이상

### 미스터리 템플릿

- **7개의 노드**와 여러 갈래의 분기가 미리 구성되어 있습니다.
- 저택에서 단서를 찾아 수수께끼를 풀어가는 미스터리 시나리오 틀을 제공합니다.
- 기본 캐릭터 1명("집사")과 씬 1개("저택")가 포함됩니다.
- 기본 설정: 미스터리 장르, 중간 플레이 시간, 중급 난이도, 12세 이상

<Note>
템플릿에 포함된 캐릭터와 씬은 시작점일 뿐입니다. 자유롭게 수정하거나 새 캐릭터와 씬을 추가하세요.
</Note>

## 시나리오 제출 요건

시나리오를 심사에 제출하려면 다음 요건을 충족해야 합니다.

| 항목 | 요건 |
|------|------|
| 노드 수 | 최소 3개 ~ 최대 200개 |
| 엔딩 | 최소 1개 ~ 최대 20개 |
| 제목 | 3자 ~ 100자 |
| 설명 | 10자 ~ 500자 |

이 외에도 시스템이 자동으로 검사하는 항목:

- **연결되지 않은 노드**: 어디에도 연결되지 않아 도달할 수 없는 노드가 있으면 경고가 표시됩니다.
- **막다른 노드**: 엔딩이 아닌데 다음 노드가 연결되지 않은 노드가 있으면 경고가 표시됩니다.
- **금칙어 검사**: 심사 제출 시 자동으로 금칙어 포함 여부를 검사합니다.

<Warning>
심사 제출은 **하루 최대 10회**, 시나리오 새로 만들기는 **1시간에 최대 5회**까지 가능합니다.
</Warning>

## 시나리오 제작 팁

1. **첫 장면에 힘을 주세요** - 이용자가 처음 보는 장면이 가장 중요합니다. 흥미를 끌 수 있는 상황이나 대사로 시작하세요.
2. **분기를 활용하세요** - 선택에 따라 이야기가 달라지면 이용자가 여러 번 플레이하게 됩니다.
3. **채팅 모드를 적절히 섞으세요** - 비주얼 노벨 진행과 AI 자유 대화 구간을 번갈아 배치하면 몰입감이 높아집니다.
4. **씬으로 장면을 정리하세요** - 같은 장소에서 벌어지는 장면들을 하나의 씬으로 묶으면 배경과 BGM 관리가 편해집니다.
5. **배경 시간 필터를 활용하세요** - 하나의 배경 이미지로 낮/밤/석양 분위기를 모두 연출할 수 있습니다.
6. **히든 엔딩을 만들어보세요** - 특수 조건을 만족해야 볼 수 있는 숨겨진 엔딩은 이용자의 탐구 욕구를 자극합니다.
7. **검증 도구를 활용하세요** - 에디터의 검증 기능이 연결 끊김, 막다른 노드 등을 자동으로 찾아줍니다. 제출 전에 꼭 확인하세요.

## 제작 후 다음 단계

시나리오 작성이 완료되면:

1. **검증 확인**: 에디터의 경고 메시지를 모두 해결합니다.
2. **프리뷰**: 직접 플레이하면서 흐름을 확인합니다.
3. **공개 설정**: 연령 등급, 장르, 태그 등을 알맞게 설정합니다.
4. **심사 제출**: 심사를 제출하면 관리자가 검토한 뒤 승인 또는 반려합니다.

<Note>
승인된 시나리오를 수정하면 **자동으로 비공개 + 작성 중 상태로 복귀**합니다. 수정 후 다시 심사를 제출해야 합니다.
</Note>

캐릭터를 시나리오에 추가하고 관리하는 방법은 [캐릭터-시나리오 연결 가이드](/mintlify/scenario-character-integration)를 참고하세요.
